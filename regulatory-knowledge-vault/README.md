# My Dev Brain

一个基于 Obsidian 的个人 LLM Wiki，用来沉淀知识、项目、个人上下文，以及系统对“我最近在变化什么”的梦境页。

它不是一个普通笔记仓库。目标是让 LLM 不只是回答问题，而是逐步理解我的知识、目标、偏好和工作方式，成为一个长期协作的外部记忆层。

## 这个项目在做什么

这个仓库把个人知识管理拆成几层：

- 原始资料层：保存文章、网页、HTML、摘录等来源材料
- 主题知识层：把零散材料沉淀成长期维护的主题页
- 项目层：跟踪正在推进的长期事项
- 个人上下文层：记录“我是谁”“我的偏好”“我的目标”
- 梦境层：定期综合最近变化，生成系统对“我的新理解”

核心思路不是“把资料喂给模型”，而是“让模型持续维护一套结构化 Wiki”。

## 适合谁

适合正在做这些事的人：

- 用 Obsidian 管理学习和项目
- 希望 LLM 越来越懂自己的上下文
- 不想只做临时问答，而想沉淀长期知识
- 想把知识管理升级成个人认知操作系统

## 怎么使用

最常见的使用方式有 4 类：

### 1. 导入新资料

当学到一篇新文章、网页或本地文件时：

- 先在 `01-资料库/` 创建资料页
- 提炼 3 到 5 条核心观点
- 标出这篇资料会影响哪些主题页

参考模板：

- [资料模板](./90-模板/资料模板.md)

### 2. 更新主题页

当某个概念开始稳定成型时：

- 不停留在资料页
- 把内容合并到 `02-主题/` 的主题页
- 写清楚定义、当前结论、关键问题、争议和下一步

### 3. 跟踪项目

当某个方向变成持续推进的事项时：

- 在 `06-项目/` 建立项目页
- 跟踪目标、进展、里程碑、问题和下一步

参考模板：

- [项目模板](./90-模板/项目模板.md)

### 4. 做一次梦

“做梦”不是单纯总结资料，而是让系统综合最近的学习、项目和个人上下文，生成一篇关于“我最近在变化什么”的页面。

触发方式：

- 在协作对话里说“做一次梦”
- 或“做一次关于我的梦”

梦境页会：

- 读取首页、个人页、项目页、日志页和近期主题页
- 区分“观察”和“推断”
- 自动落到 `04-综合/梦境/`
- 不直接改写稳定的个人页面

参考：

- [如何做梦](./00-首页/如何做梦.md)
- [梦境目录](./04-综合/梦境/README.md)
