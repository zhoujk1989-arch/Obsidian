# 存款保险系统-表间关系
> 页面定位：存款保险2.0系统报表间的数据校验、关联规则和业务逻辑  
> 最后更新：2026-05-14

## 表间关系总览

存款保险2.0系统当前已入库的两张报表形成**一对多关系**：

```
存款人信息表 (PDPD01_CKRXX)
  ↓ 1:N 关联关键字段：identi_code
  ↓ 数据聚合方向
存款账户信息表 (PDPD01_CKZHXX)
```

| 维度 | 存款人表(CKRXX) | 存款账户表(CKZHXX) |
|------|------------|------------|
| 报表定位 | 存款人维度的汇总表 | 存款账户维度的明细表 |
| 物理表名 | PDPD01_CKRXX | PDPD01_CKZHXX |
| 粒度 | 按存款人聚合 | 账户级明细 |
| 字段数 | 23 | 30 |
| 核心主键 | identi_code（存款人识别码） | id（账户ID）+identi_code |
| 数据关系 | 聚合维度 | 明细维度 |

## 关键字段关联

### 识别码关联(identi_code)

两表通过**识别码(identi_code)**进行主要关联。

| 字段 | 数据类型 | 说明 |
|------|---------|------|
| CKRXX.identi_code | varchar(50) | 同一存款人的唯一标识（投保机构内） |
| CKZHXX.identi_code | varchar(50) | 同一存款人的唯一标识，应与CKRXX一致 |

**关联规则：**
- 同一存款人可能在CKZHXX中有多条记录（多个账户）
- 所有该存款人账户的记录中，identi_code值应完全一致
- 若同一存款人的多个客户号未归户到同一identi_code，数据视为违反填报要求

### 金额层级关联

存款人表的**三层金额**都是从账户表逐行本息合计聚合而来：

#### 第一层：各存款账户本息合计

```
CKRXX.deposit_amount 
  = Σ(CKZHXX.deposit_amount) 
  where identi_code = CKRXX.identi_code
```

**含义：** 同一存款人在投保机构的全部账户余额合计

**验证方式：**
1. 按CKRXX.identi_code值筛选CKZHXX所有行
2. 将这些行的deposit_amount列求和
3. 比对是否等于CKRXX.deposit_amount

#### 第二层：被保险存款本息合计

```
CKRXX.insured_dep_amount 
  = Σ(CKZHXX.deposit_amount) 
  where identi_code = CKRXX.identi_code 
    AND acct_safe_flag = '1'
```

**含义：** 该存款人被纳入存款保险制度覆盖的账户余额合计

**验证方式：**
1. 在上述筛选基础上，再加条件 `acct_safe_flag='1'`
2. 对满足条件的行的deposit_amount求和
3. 比对是否等于CKRXX.insured_dep_amount

**关键业务规则：** `CKRXX.insured_dep_amount ≤ CKRXX.deposit_amount`

#### 第三层：受保存款本息合计

```
CKRXX.inner_insured_dep_amount 
  = min(50万元, CKRXX.insured_dep_amount)
```

**含义：** 被保险存款中在50万元最高偿付限额内的部分

**验证方式：**
1. 确认 `CKRXX.inner_insured_dep_amount ≤ CKRXX.insured_dep_amount`
2. 确认 `CKRXX.inner_insured_dep_amount ≤ 50万`

## 数据结构维度的对应关系

### 产品类别维度

账户表在产品类别(dep_pro_type)维度进行**穿透**（每条记录一个产品类别），存款人表则对所有产品类别进行**汇总**。

| 账户表(CKZHXX) | 存款人表(CKRXX) | 说明 |
|------------|------------|------|
| 单条记录中：dep_pro_type=02(定期存款) | 汇总所有dep_pro_type的余额 | 账户表：明细 → 存款人表：汇总 |

**按产品类别对账：**
可按dep_pro_type对账户表进行分组，验证各产品类别的汇总值：
```sql
SELECT 
  dep_pro_type, 
  SUM(deposit_amount) as total_by_product
FROM CKZHXX
WHERE identi_code = 'XX'
GROUP BY dep_pro_type
```

### 账户类型维度

类似地，账户表在账户类型(dep_acc_type)维度进行穿透，存款人表在所有账户类型汇总。

| 账户表(CKZHXX) | 存款人表(CKRXX) | 说明 |
|------------|------------|------|
| 单条记录中：dep_acc_type=01(Ⅰ类户) | 汇总所有dep_acc_type的余额 | 账户表：明细 → 存款人表：汇总 |

**按账户类型对账：**
```sql
SELECT 
  dep_acc_type, 
  SUM(deposit_amount) as total_by_acc_type
FROM CKZHXX
WHERE identi_code = 'XX'
GROUP BY dep_acc_type
```

## 存款保护标识逻辑

### acct_safe_flag 字段说明

`acct_safe_flag`（是否受存款保险制度保护）是账户级的**二值标识**，决定了该账户是否纳入被保险存款统计。

| 值 | 含义 | 对应计算 |
|----|------|--------|
| '1' | 账户纳入保险覆盖 | 该账户的deposit_amount计入CKRXX.insured_dep_amount |
| '0' | 账户不纳入保险覆盖 | 该账户的deposit_amount不计入CKRXX.insured_dep_amount |

### 赋值规则（待确认）

根据《存款保险条例》，通常以下账户**不纳入保险覆盖**（待原文确认）：
- 某些特殊存款产品（如部分结构性存款、部分协议存款）
- 非金融机构客户的某些账户

**待确认事项：**
- 确切的acct_safe_flag=0的判断标准
- 是否存在某个产品类别的所有账户都不纳入保险
- 是否按照存款金额超过50万就自动标记acct_safe_flag=0（应该不是，因为内部计算会自动限额）

## 数据一致性校验规则

### 强制性约束

这些约束**必须满足**，否则数据存在错误或违反填报规范：

| 校验项 | 公式/条件 | 等级 | 说明 |
|--------|---------|------|------|
| 金额约束1 | CKRXX.insured_dep_amount ≤ CKRXX.deposit_amount | 强制 | 被保险存款不能超过总存款 |
| 金额约束2 | CKRXX.inner_insured_dep_amount ≤ CKRXX.insured_dep_amount | 强制 | 受保存款不能超过被保险存款 |
| 金额约束3 | CKRXX.inner_insured_dep_amount ≤ 50万 | 强制 | 最高偿付限额约束 |
| 汇总约束1 | Σ(CKZHXX.deposit_amount \| identi_code=X) = CKRXX.deposit_amount | 强制 | 账户汇总必须等于人表汇总 |
| 汇总约束2 | Σ(CKZHXX.deposit_amount \| identi_code=X & acct_safe_flag='1') = CKRXX.insured_dep_amount | 强制 | 被保险账户汇总必须一致 |
| 明细约束 | CKZHXX.deposit_amount = CKZHXX.principal + CKZHXX.interest | 强制 | 账户表的本息计算 |

### 检查性约束

用于发现潜在问题或异常数据：

| 检查项 | 条件 | 优先级 | 说明 |
|--------|------|--------|------|
| 零余额账户 | deposit_amount = 0 | 低 | 应确认是否为正常状态（休眠、已结清等） |
| 高金额账户 | deposit_amount > 500万 | 低 | 对于大额存款应逐笔确认 |
| 异常状态 | dep_status not in ('01','02','03','04') | 高 | 存款状态应在允许值范围内 |
| 外币处理 | 涉及外币账户 | 中 | 确认汇率折算时间是否正确 |

## 对账流程

### 标准对账顺序

1. **第一步：粒度对账**
   - 统计CKZHXX中identi_code=X的记录数
   - 核对账户维度的本息合计逐笔正确

2. **第二步：总额对账**
   - 按公式验证 Σ(CKZHXX.deposit_amount) = CKRXX.deposit_amount

3. **第三步：保险标识对账**
   - 核对acct_safe_flag的赋值规则
   - 验证 Σ(CKZHXX.deposit_amount | acct_safe_flag='1') = CKRXX.insured_dep_amount

4. **第四步：限额对账**
   - 验证 CKRXX.inner_insured_dep_amount = min(50万, CKRXX.insured_dep_amount)

5. **第五步：状态对账**
   - 检查verify_status和data_status是否一致

### 常见对账问题排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|--------|--------|
| 总额不符 | 账户表某些行缺失或金额错误 | 重新统计CKZHXX，逐行对账 |
| 被保险存款不符 | acct_safe_flag赋值错误 | 检查每行的acct_safe_flag值 |
| 受保存款不符 | 上游系统计算insured_dep_amount时未限额50万 | 确认是否有 >50万的被保险存款被计算在内 |
| 产品/账户类型不齐 | 某些产品或账户类型的账户缺失 | 按dep_pro_type/dep_acc_type分组统计 |
