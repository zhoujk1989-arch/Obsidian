## 报表定位

本表是存款保险2.0系统中**存款账户维度的明细表**，与存款人信息表(PDPD01_CKRXX)通过识别码关联，一对多关系。同一存款人可能拥有多个不同产品类别、账户类型的存款账户，本表在账户级别记录每个账户的产品类型、余额、保护状态等。

**主要承载监管指标：**
- 存款账户产品分类与账户分类（10类产品 × 3类账户 = 30种组合）
- 账户级本息合计金额
- 开户信息与联系方式
- 存款保护状态标识
- 账户状态管理

## 表间关系

```
存款人信息表 (PDPD01_CKRXX)
  ↓ 1:N 按识别码聚合自
存款账户信息表 (PDPD01_CKZHXX)
  
关键关联字段：存款人信息.identi_code = 存款账户信息.identi_code

数据验证逻辑：
- PDPD01_CKRXX.deposit_amount = Σ(PDPD01_CKZHXX.deposit_amount where identi_code='X')
- PDPD01_CKRXX.insured_dep_amount = Σ(PDPD01_CKZHXX.deposit_amount where identi_code='X' AND acct_safe_flag='1')
- PDPD01_CKRXX.inner_insured_dep_amount = min(50万, PDPD01_CKRXX.insured_dep_amount)
```

## 报表填报规则

### 存款产品与账户类型排序

**同一存款人的账户应按以下顺序排列：**
1. 先按10个**存款产品类别**排序（01-10）
2. 再在同一产品类别内按3个**账户类型**排序（Ⅰ类户/Ⅱ类户/Ⅲ类户）
3. "其他"账户类型需以"其他-具体账户类型"形式注明

**示例排序：**
```
存款人A：
- 活期存款-Ⅰ类户 (01, 01)
- 活期存款-Ⅱ类户 (01, 02)
- 活期存款-Ⅲ类户 (01, 03)
- 定期存款-Ⅰ类户 (02, 01)
- ...
- 其他存款-其他 (10, 其他-具体类型)
```

### 账户号码穿透规则

- 存款账户号码应**穿透至最底层子账户**
- 例如一卡通下挂多个子账户（活期、定期等）应分别填写各自的子账户号
- 每条记录对应一个最小粒度的存款账户

### 金额填报规则

- **本金、利息、本息合计均不能填写短杠**
- 外币存款统一按中国外汇交易中心每月最后交易日汇率中间价折算成人民币
- 数据精度：四舍五入保留至小数点后两位

### 存款保护标识规则

- `acct_safe_flag`标识该账户是否纳入存款保险制度覆盖
- 该字段与存款人表的`insured_dep_amount`和`inner_insured_dep_amount`的逻辑关联
- 根据产品特征（如大额存单通常不纳入）和金额确定

### 零余额账户处理

- 零余额（principal=0, interest=0, deposit_amount=0）的账户需填报
- 即使余额为零，仍需完整记录账户信息

## 与其他表的关系对比

### 与存款人信息表(PDPD01_CKRXX)的关系

| 维度 | 存款人表 | 存款账户表 |
|------|---------|---------|
| 主键 | identi_code + 存款人维度 | identi_code + 账户ID |
| 粒度 | 存款人维度（汇总） | 账户维度（明细） |
| 字段数 | 23 | 30 |
| 金额 | 汇总后的三层分层金额 | 账户级本息合计 |
| 数据流向 | 存款人表聚合自账户表 | 原始明细数据 |
| 关键校验 | 汇总逻辑 | 个别账户详情 |

**推荐的数据验证流程：**
1. 先核对存款账户表(PDPD01_CKZHXX)的账户数量与结构
2. 按账户汇总本息合计，逐个验证与存款人表的一致性
3. 按acct_safe_flag标识的账户计算被保险存款本息合计
4. 与存款人表的insured_dep_amount比对

## 监管意义

1. **存款保险覆盖确认**：通过acct_safe_flag逐账户标识受保情况
2. **结构性分析**：按产品类别与账户类型交叉分析存款结构
3. **风险分层**：结合账户状态(正常/休眠/合同限制/司法限制)识别风险账户
4. **保费计算支撑**：提供账户维度的本息明细，支撑被保险存款本息合计的逐笔验证
5. **数据可追溯**：存储开户证件、联系方式等基础信息，便于后续核查与沟通
