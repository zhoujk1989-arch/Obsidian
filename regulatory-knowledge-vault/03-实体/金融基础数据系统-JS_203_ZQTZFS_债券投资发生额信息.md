# 金融基础数据系统-JS_203_ZQTZFS_债券投资发生额信息

## 简介

JS_203_ZQTZFS 是金融基础数据系统首张债券投资维度报表，采集报告期内债券买入、卖出、买入又卖出的发生额数据，以债券投资交易为单位逐笔报送。报表编码延续 JS_203 系列，为存款大类下的投资类细类。

## 当前结论

- JS_203_ZQTZFS 是金融基础数据系统首张债券投资维度报表，也是首张 JS_203 系列报表。
- 22个字段覆盖债券投资交易全要素：主体标识（金融机构代码/内部机构号）、债券标识（债券代码/总托管机构/品种/信用级别）、发行人信息（证件代码/地区/行业/企业规模/经济成分/国民经济部门）、交易要素（交易日期/流水号/买入卖出标志）、金额利率（成交金额/折人民币/票面利率）、日期要素（债权债务登记日/起息日/兑付日期）。
- 报送粒度：以单笔交易流水为核心识别维度逐笔报送。
- 与 JS_201 系列（贷款类）和 JS_202 系列（存款类）形成"贷款+存款+投资"三维度覆盖，金融基础数据系统从此覆盖银行三大核心资产业务线。
- 债券品种码值覆盖范围极广：国债（凭证式/储蓄式/记账式）、地方政府债、央行票据、政策性银行债、商业银行各类资本债、非金融企业债务融资工具（MTN/CP/SCP/PPN/SMECN）、资产支持票据/证券、企业债券/公司债/可转债等，共约34类。
- 债券信用级别码值采用人民银行信用评级管理指导意见标准，覆盖 AAA 至 D 共25级（含 A-1/A-2/A-3 短期评级），多评级采取孰低原则。

## 重点口径

1. **债券投资发生额定义**：报告期内买入、卖出、买入又卖出的债券。以债券投资交易为单位逐笔报送，不是余额维度。
2. **折人民币必填**：成交金额折人民币字段为必填项，外币交易按金融机构会计核算使用汇率折算，人民币交易直接填人民币金额。
3. **多评级孰低原则**：债券信用级别多评级时采取孰低原则；出现其他评级体系时填符合人民银行指导意见的评级；仅有其他评级体系时填C99；无评级填C00。
4. **发行人证件代码分层规则**：境内机构→统一社会信用代码/组织机构代码；境内个人→有效身份证件；资管产品→资管产品统计编码；境外机构→LEI码；境外个人→金融机构自设编码。
5. **地区代码区分境内外**：境内用GB/T2260县区级码；境外前3位000+GB/T2659国别码，港澳台按境外客户处理，不得填报810000/820000等行政区划代码。

## 重点规则

1. **债券总托管机构唯一性**：对于单只债券仅存在唯一的总托管机构，300/400/900/000四选一。
2. **票面利率格式**：采用百分数表示，如5.2%填5.20000（5位小数），无票面利率可不填报。
3. **企业规模分类**：非金融企业按《统计上大中小微型企业划分办法（2017）》；金融企业按银发〔2015〕309号文；资产支持证券填报发起人企业规模；非企业类单位必须填CS05。
4. **经济成分分层**：债券品种为资产支持证券的填报发起人经济成分，境外发行机构无需填写。
