## 报表定位

本表是存款保险2.0系统中记录存款人基本信息与存款余额分层的核心表。通过同一存款人识别码，将个人存款人的身份信息、存款产品类型、账户信息与受保情况进行归户汇总，为存款保险保费计算和风险管理奠定数据基础。

**主要承载监管指标：**
- 存款人识别与归户（同一存款人下的多账户合并）
- 各存款账户本息合计（存款存量）
- 被保险存款本息合计（纳入存款保险范围金额）
- 受保存款本息合计（50万最高偿付限额内金额）
- 外币存款折人民币管理

  - `inner_insured_dep_amount` = min(50万, insured_dep_amount)

外币存款统一按中国外汇交易中心每月最后一个交易日汇率中间价折算成人民币。

## 报表填报规则

### 存款人识别码（F7 identi_code）

**关键规则：**
1. 同一投保机构内，同一存款人应有唯一识别码
2. 若同一存款人有多个客户号，应**归户合并**至同一识别码下
3. 对于非实名账户、久悬账户、应解汇款、汇出汇款等特殊账户，若勤勉尽职后仍无法穿透至具体存款客户，可分别按类型作为同一客户

**校验关系：** 存款人信息表的识别码 = 存款账户信息表的识别码（两表间的主键关联）

### 存款产品类别排序

存款人名下各存款账户应按以下10个存款产品类别**依序排列**（暗示本表可能涉及多行记录每个存款人）：
1. 活期存款
2. 定期存款
3. 定活两便存款
4. 通知存款
5. 协议存款
6. 协定存款
7. 保证金存款
8. 结构性存款
9. 大额存单
10. 其他存款

### 存款账户穿透

- 存款账户号码应穿透至**最底层子账户**
- 例如一卡通下挂的活期、定期等多个子账户应分别填写

### 存款账户类型排序

各存款账户应按以下4类**依序排列**：
1. Ⅰ类户
2. Ⅱ类户
3. Ⅲ类户
4. 其他（其他+具体账户类型）

### 汇总逻辑

- **存款人姓名字段**：按存款人统计合计数
- **存款账户号码字段**：按存款账户统计合计数
- **其他汇总项**（各金额）：直接进行数值汇总
- **数据精度**：四舍五入保留至小数点后两位

### 特殊账户处理

- 零余额存款账户需填报（不能跳过）
- 开户证件类型覆盖13类，包括身份证、护照、港澳台证件等

## 与其他表的关联

**存在关联的表：**
- **存款账户信息表 (PDPD01_CKZHXX)**：通过识别码 `identi_code` 关联；存款账户信息表维度更细，本表是其存款人维度的汇总
- **存款保险制度覆盖规则**：根据《存款保险条例》，被保险存款本息合计与受保存款本息合计的层级关系反映的是保险覆盖的等级管理

**金额对应关系：**
```
各存款账户本息合计 (deposit_amount)
  ├── 纳入保险范围的部分 = 被保险存款本息合计 (insured_dep_amount)
      └── 最高偿付限额内的部分 = 受保存款本息合计 (inner_insured_dep_amount) ≤ 50万
  └── 超出保险范围的部分（保险外）
```

## 监管意义

1. **存款保险保费计算基数**：被保险存款本息合计是保费计算的直接基础
2. **存款人风险分层**：受保存款本息合计反映了在保险覆盖范围内的风险敞口
3. **归户合并管理**：同一存款人多账户的识别码汇总是存款保险制度的核心要求
4. **时间序列监测**：按季报送，能动态追踪每个存款人的存款变动趋势
