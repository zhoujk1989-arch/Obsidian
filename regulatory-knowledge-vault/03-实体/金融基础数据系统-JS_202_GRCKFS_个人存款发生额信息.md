# 金融基础数据系统-JS_202_GRCKFS_个人存款发生额信息

## 简介

- 所属系统：金融基础数据系统

- 报表名称：个人存款发生额信息

- 报表细类：普通存款
- 当前原文依据：[[01-资料库/金融基础数据系统/2026-05-04-JS_202_GRCKFS_个人存款发生额信息-原文|原文页]]
- 知识沉淀位置：本页直接维护该报表的提炼结论、重点规则和常见口径

## 当前结论

- JS_202_GRCKFS 是金融基础数据系统首张个人存款发生额维度报表，20字段，采集金融机构报告期内吸收、到期、吸收又到期的个人存款。
- 与 JS_202_CLTYCK（存量同业存款信息）形成"存款主体"区分：CLTYCK 采集同业存款（机构间），GRCKFS 采集个人存款（自然人）。
- 与 JS_202_FTYDWC（存量非同业单位存款信息）形成"个人 vs 单位"互补：FTYDWC 采集非同业单位存款（对公），GRCKFS 采集个人存款（对私）。
- 与 JS_202_DWCKFS（非同业单位存款发生额信息）形成"个人 vs 单位"互补：DWCKFS 采集非同业单位存款发生额（对公），GRCKFS 采集个人存款发生额（对私）。
- 金融基础数据系统存款大类完整覆盖：同业存量（JS_202_CLTYCK）+ 同业发生额（JS_202_TYCKFS）+ 非同业单位存量（JS_202_FTYDWC）+ 非同业单位发生额（JS_202_DWCKFS）+ 个人存款发生额（JS_202_GRCKFS）。
- 报送粒度：以单笔交易流水为核心识别维度逐笔报送，以存款账户下单笔存款协议为单位。

## 重点口径

1. **个人存款定义**：指金融机构报告期内吸收、到期、吸收又到期的个人存款。以存款账户下单笔存款协议为单位逐笔报送。
2. **折人民币必填**：发生金额折人民币为必填字段，外币交易按金融机构会计核算使用汇率折算，人民币交易直接填写。
3. **利率填报规则**：填报年利率，只填数字，不需填写百分号"%"。
4. **通知存款到期日特殊规则**：一天、七天提前通知存款的协议到期日分别填写"1999-01-01"、"1999-01-07"。
5. **活期到期日默认值**：业务类型为活期等业务没有到期日的，默认为"9999-12-31"。
6. **客户证件加密**：个人身份证件代码根据人民银行统一加密规则加密报送。
7. **境外客户地区代码**：客户注册地为港澳台的，应按境外客户填报地区代码，不得填报行政区划代码如810000、820000等。
8. **存款产品类别**：必须填写最底层类别，覆盖活期储蓄（D013）、定期储蓄（D014）、结构性存款（D08）、大额存单（D16）等个人常用产品。

## 重点规则

1. **逐笔报送**：以单笔交易流水为核心识别维度逐笔报送，非汇总报送。
2. **客户身份识别**：境内个人填报个人有效身份证件代码；境外个人填报金融机构自行设定的唯一编码或LEI码（如适用）。
3. **地区代码分级**：境内客户用GB/T2260县区级代码；境外客户用000+GB/T2659三位国别码。
4. **产品类别码值**：D013（活期储蓄存款）和D014（定期储蓄存款）为个人存款核心产品类别；D08（结构性存款）、D091（贷记卡存款）、D092（准贷记卡存款）等也涉及个人客户。

| 个人存款是否包含单位存款？ | 不包含。GRCKFS 仅采集个人存款（自然人），单位存款由 FTYDWC（存量）和 DWCKFS（发生额）采集 | 本表报送范围定义 |
