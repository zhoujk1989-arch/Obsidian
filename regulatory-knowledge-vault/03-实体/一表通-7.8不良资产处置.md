# 一表通-7.8不良资产处置

## 简介

一表通系统交易信息域中的不良资产处置明细层报表，采集上报机构对不良资产的处置及核销后收回情况。

## 关键信息

- 所属系统：一表通系统
- 报表编码：7.8
- 报表名称：不良资产处置
- 当前原文依据：[[01-资料库/一表通系统/2026-04-28-7.8不良资产处置-填报说明-原文]]
- 知识沉淀位置：本页直接维护该报表的提炼结论、重点规则和常见口径

## 当前结论

1. 本表是一表通系统的新增报表（1.0版首次纳入），用于采集不良资产处置全生命周期数据。
2. 填报范围包括两类业务：处置类（核销前处置）和收回类（核销后收回），两类业务字段互斥——处置业务中收回字段可置空，收回业务中处置字段可置空。
3. 处置类型码值体系覆盖 9 大类：向上级行划转、重组上调、现金清收、资产转让（5子类）、核销（6子类）、以物抵债（2子类）、其他、已核销收回。
4. 资产类型码值体系覆盖 8 类资产 + 银行自定义：个人贷款、对公贷款、信用卡贷款（对公/个人）、债券、资管产品、其他债权融资类产品、股权。
5. 处置状态码值 4 类：账存案存、账存案销、账销案存、完全终结，反映不良资产处置与债务关系存续的组合状态。
6. 与 G12 报表存在口径关联：资产类型中的债券、资管产品、其他债权融资类产品分别与 G12 附注项"15.2 债券处置总额""15.3 资管产品处置总额""15.4 其他债权融资类产品处置总额"范围一致。

### 处置与收回互斥规则

- **处置业务**：处置字段（G080008~G080012）填报，收回字段（G080013~G080020）可置空
- **收回业务**：收回字段填报，处置字段可置空
- 报数前已完成处置核销的，不填报处置核销情况，但报送核销后收回现金情况

### 打包资产拆分规则

处置资产如果是打包的，本金和利息金额（G080009/G080010/G080011）按比例拆分填报。

### 员工 ID 填报规则（G080025）

- 需人工发起处置方案的（资产转让、核销、以物抵债等）：填报发起处置方案的员工 ID
- 不涉及人工发起处置流程的（现金清收、向上迁徙等）：填报管户客户经理员工 ID
- 已核销收回业务可置空

### 行业类型（G080024）

- 优先按贷款投向行业划分，不掌握时按贷款客户所属行业划分
- 按《国民经济行业分类》（GB/T 4754-2017）标准
- 境外贷款按"99999"填报
- 对公贷款和个人经营性贷款必填，个人非经营性贷款可空

### 币种（G080022）

- 遵循 GB/T 12406-2008，三位字母编码（如 CNY）
- 多币种合计折算为人民币的，填报 BWB

## 重点规则

1. 协议ID（G080004）为强制性非空校验字段，银行机构需保证机构内部针对同一客户有且仅有一个客户ID。
2. 转让资产相关字段（G080016/G080017）仅在通过资产转让形式处置时填报，不涉及可置空。
3. 处置日期默认值 9999-12-31，收回业务可置空。
4. 处置收回日期默认值 9999-12-31，处置业务可置空。
