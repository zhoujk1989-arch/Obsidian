# 存款保险2.0系统

## 系统定位

存款保险2.0系统是中国银保监会为落实《存款保险条例》建立的监管数据采集与管理平台。通过该系统，投保机构（主要为商业银行等存款接收金融机构）按季向监管部门报送存款人与存款账户的详细信息，便于监管部门监测存款保险基金风险、优化保费计算、评估制度覆盖效果。

**制度背景：** 2015年5月1日《存款保险条例》正式实施，建立存款保险制度。存款保险实行限额偿付，最高偿付限额为50万元/存款人/投保机构。系统2.0升级体现了数据精细化管理的演进。

## 系统功能

1. **存款人信息采集**：汇总每个存款人的身份、账户、存款余额信息
2. **存款账户穿透**：记录每个账户的产品类别、账户类型、存款余额分层
3. **保险覆盖监测**：追踪被保险存款与受保存款范围
4. **保费计算支撑**：被保险存款本息合计是保费计算的直接基础
5. **风险评估**：动态监测单个投保机构的存款结构与保险敞口

## 关键数据指标

### 存款人维度

| 指标 | 字段名 | 口径 | 单位 | 说明 |
|------|--------|------|------|------|
| 各存款账户本息合计 | `deposit_amount` | 同一存款人所有账户合计 | 元 | 存款总规模 |
| 被保险存款本息合计 | `insured_dep_amount` | 纳入保险范围金额 | 元 | 保费计算基数 |
| 受保存款本息合计 | `inner_insured_dep_amount` | 50万限额内部分 | 元 | 实际保险覆盖额度 |

### 存款结构维度

- **按存款产品类别**：活期、定期、大额存单、结构性存款等10类统计
- **按账户类型**：Ⅰ类户（全功能）、Ⅱ类户（有限功能）、Ⅲ类户（小额账户）、其他
- **按币种**：人民币、外币（折算中间价）
- **按状态**：正常、休眠、合同限制、司法限制

## 系统参数与规则

### 报送频率与范围

- **报送频率**：季度（通常在报告期结束后一定时间内报送）
- **覆盖范围**：所有在中国境内的已注册投保机构（主要为商业银行）
- **数据日期**：YYYYMMDD，精确到报告期最后一日

### 识别码规则

**同一存款人的唯一标识：**
- 由投保机构自定义（不是全国统一编码）
- 若同一存款人有多个客户号，必须归户合并至同一识别码
- 对于非实名账户、久悬账户等无法穿透的账户，可按类型分别标记

**识别码关联：** 存款人信息表.identi_code = 存款账户信息表.identi_code

### 存款金额计算规则

**分层逻辑：**
```
总存款 (deposit_amount)
  = 纳入保险范围 (insured_dep_amount)
      + 超出保险范围
  
纳入保险范围 (insured_dep_amount)
  = 最高限额内 (inner_insured_dep_amount ≤ 50万)
      + 超过限额部分 (不受保)
```

**外币处理：** 按中国外汇交易中心每月最后一个交易日的汇率中间价折算成人民币

**四舍五入：** 所有金额字段保留至小数点后两位

### 状态转换

**数据状态流程：** 待处理(td) → 待提交(uc) → 提交待审核(ac) → 审核通过(ap) ✓ / 审核未通过(af) ✗

**校验状态流程：** 未校验(uv) → 校验中(vc) → 校验通过(vp) ✓ / 校验通过警告(vpw) ⚠ / 校验不通过(vf) ✗

## 参考标准与依据

- **《存款保险条例》**（2015年5月1日正式实施）
  - 制度框架、偿付限额、保费基数
  - 最高偿付限额：50万元/存款人/投保机构

- **《存款统计分类及编码》（JR/T0134—2016）**
  - 存款状态分类（正常/休眠/合同限制/司法限制）
  - 存款产品类别编码

- **《商业银行存款保险保费收取办法》**
  - 被保险存款本息合计的计算与确认
