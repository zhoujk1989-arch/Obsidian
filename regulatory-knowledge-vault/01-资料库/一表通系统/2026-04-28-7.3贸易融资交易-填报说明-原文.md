## 字段填报说明原文

| 序号 | 数据项编码 | 数据项名称 | 填报说明 |
| --- | --- | --- | --- |
| F1 | G030001 | 交易ID | 按照既定规则生成，用于唯一识别以货币及服务为媒介的价值交换的交易编码，包括但不限于信贷、票据、投资等。生成规则为：采用长度不超过100位数字与字母组合，不足部分不用补齐。 |
| F2 | G030002 | 协议ID | 按照既定规则生成用于区分客户的唯一编码。生成规则为：采用长度不超过60位数字与字母组合，不足部分不用补齐。银行机构需保证机构内部针对同一客户有且仅有一个客户ID。 |
| F3 | G030003 | 分户账号 | 银行机构内具有唯一区分性、用于逐笔记录交易明细或用于总账核对的明细核算的最细一级账号。客户是境内涉密法人的，填写"*********"。 |
| F4 | G030004 | 客户ID | 按照既定规则生成用于区分与金融消费者或第三方发生金融业务及其他民事法律关系约定的条件，如合同、票据等的编码。生成规则为：采用长度不超过60位数字与字母组合，不足部分不用补齐。 |
| F5 | G030005 | 交易机构ID | 按照既定规则生成分配给实际业务经办最小单位机构的唯一编码。生成规则为：使用长度不超过24位数字与字母组合，前11位与金融许可证号前11位相同，后13位使用内部机构号，不足部分不用补齐。理财登记中心填报机构ID统一使用金融许可证。如果发行地涉及多个国家，机构ID填写共同上级行机构ID，可以支持多地区填报。 |
| F6 | G030006 | 核心交易日期 | 在核心系统上发生该交易的日期，特定日历日的标识。由日历年、日历月、日历日等数据组合表示。默认值9999-12-31。 |
| F7 | G030007 | 核心交易时间 | 在核心系统上发生该交易的时间，默认值00:00:00。 |
| F8 | G030008 | 交易类型 | 01 发放 此笔交易为贸易融资放款。<br>02 收回 此笔交易为收回贸易融资本金。利随本清的业务填报。<br>03 收息 此笔交易为利息收回。<br>00-XX 其他-银行自定义 无法以枚举类型填报的，以"00-XX"填报，其中"XX"为银行自定义类型。<br><br>备注：<br>01-发放：此笔交易为贸易融资放款。<br>02-收回：此笔交易为收回贸易融资本金。利随本清的业务填报。<br>03-收息：此笔交易为利息收回。<br>04-其他：上述交易类型以外的其他交易。 |
| F9 | G030009 | 交易金额 | 填报交易的发生额。数值可为负值。交易方向由借贷标识确定。 |
| F10 | G030010 | 币种 | 参照金融监管总局（或原银保监会）发布《公共数据标准》。遵循《表示货币和资金的代码》（GB/T 12406-2008），采用三位数字字母编码，如CNY。多个币种合计折算为人民币（本外币合计）的，填报为BWB。财务类数据应该按分币种填报。 |
| F11 | G030011 | 业务余额 | 交易发生后，此笔贸易融资业务的余额。 |
| F12 | G030012 | 对方账号 | 交易对手的银行账号。 |
| F13 | G030013 | 对方户名 | 交易对手账号对应的开户名称。 |
| F14 | G030014 | 对方账号行号 | 交易对手账号对应的银行机构代码（即12位人行支付行号），跨境交易行号可填SWIFT编码。对方账号和对方户名必须填实际收款人，不可填中间过渡账户或清算账户。涉及第三方支付平台的，可以为空。 |
| F15 | G030015 | 对方行名 | 交易对手账户所在银行机构的规范全称，与金融许可证一致。 |
| F16 | G030016 | 经办员工ID | 业务经办员工的员工ID；若为自动审批，则填写"自动"。 |
| F17 | G030017 | 授权员工ID | 授权此笔交易流水操作的员工ID。若为系统自动授权，则填写"自动"；若不涉及允许为空。 |
| F18 | G030019 | 备注 | - |
| F19 | G030018 | 采集日期 | 数据表字段信息的采集日期，特定日历日的标识。由日历年、日历月、日历日等数据组合表示。 |
