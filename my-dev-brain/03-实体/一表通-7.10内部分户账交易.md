# 一表通-7.10内部分户账交易

## 简介

采集填报机构的所有内部分户账的变动明细，根据会计核算科目，除单列账之外的科目原则上都归入内部账采集。共 31 个字段。

## 关键信息

- 所属系统：[[03-实体/一表通系统|一表通系统]]
- 报表编码：7.10
- 报表名称：内部分户账交易
- 数据域：账务信息
- 字段总数：31
- 主键：交易ID（G100001）、分户账号（G100002）、机构ID（G100029）、核心交易日期（G100003）、核心交易时间（G100004）、采集日期（G100028），共 6 个主键
- 强制性校验字段：18 个（交易ID、分户账号、机构ID、核心交易日期、核心交易时间、币种、交易类型、科目ID、科目名称、借贷标识、交易金额、经办员工ID、冲补抹标识、现转标识、交易渠道、科目ID、科目名称、采集日期）
- 当前原文依据：[[01-资料库/一表通系统/2026-04-28-7.10内部分户账交易-填报说明-原文]]
- 知识沉淀位置：本页直接维护该报表的提炼结论、重点规则和常见口径

## 当前结论

- 7.10内部分户账交易是账务信息数据域中记录内部分户账**逐笔交易流水**的报表，与 4.3 分户账信息（静态快照）、4.4 分户账变动情况（汇总变动）共同构成科目-账户两层映射体系中的"明细流水"层。
- 采集范围：填报机构的所有内部分户账的变动明细，除单列账之外的科目原则上都归入内部账采集。
- 监管反馈（20231107 汇总问题序号【00117】）：建议与 EAST 报送范围一致——单列账报送至信用卡、对公/个人等分户账中；资本账户需要报送。
- 主键由交易ID + 分户账号 + 机构ID + 核心交易日期 + 核心交易时间 + 采集日期联合构成。
- 交易金额一表通允许正值、负值和 0（由借贷标识确定方向），EAST 要求交易金额为非负值。
- 利率字段（G100011）为一表通新增字段，EAST 无对应字段。
- 会计日期（G100031）和备注（G100030）为 2.1 版新增字段。

## 重点口径

### 交易类型码值

| 码值 | 含义 |
| --- | --- |
| 01 | 转账：客户发起的账户资金转移的交易 |
| 02 | 取现：客户取出现金 |
| 03 | 存现：客户存入现金 |
| 04 | 消费：在商户消费发生的交易 |
| 05 | 代发：企事业单位委托银行发放到对应账户（如代发工资） |
| 06 | 代扣：按客户授权从客户账户扣除款项支付给第三方 |
| 07 | 代缴：客户委托银行从自己账户划转资金缴纳费用 |
| 08 | 结息：按利率和计息方式计算利息并计入/扣除 |
| 09 | 批量业务：针对多个客户、多笔交易同时处理 |
| 10 | 贷款发放：交易为本行贷款发放 |
| 11 | 还款-还本：归还贷款本金 |
| 12 | 还款-还息：归还贷款利息 |
| 13 | 银证转账：资金转入、转出证券账户 |
| 14 | 投资理财：购买本行发行或代销的投资理财产品 |
| 00-XX | 其他-银行自定义：无法以枚举类型填报的交易 |

## EAST 差异

- **码值转换**：5 个字段需码值映射（交易类型、借贷标识、交易渠道、冲补抹标识、现转标识），一表通为数字码值，EAST 为文本码值。
- **日期格式转换**：4 个日期字段需格式转换（核心交易日期、核心交易时间、进账日期、销账日期、采集日期），一表通为 YYYY-MM-DD / HH:MM:SS 格式，一表通源系统为 YYYYMMDD / hhmmss 格式。
- **机构ID拼接**：机构ID 需拼接金融许可证号前 11 位 + 内部机构号后 13 位。
- **交易金额**：一表通允许正值、负值和 0，EAST 要求非负值。
- **新增字段**：利率（G100011）为一表通新增，EAST 无对应字段；会计日期（G100031）和备注（G100030）为 2.1 版新增。
- **数据类别变更**：交易类型（G100006）和交易渠道（G100019）在 2026 年制度升级中由代码类改为文本类。

## 版本变化

- **1.0 版**：基础 29 个字段。
- **2.0 试用版**：新增机构ID（G100029）。
- **2.1 版**：新增会计日期（G100031）、备注（G100030）；交易类型和交易渠道数据类别由代码类改为文本类。
