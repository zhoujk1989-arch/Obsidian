## 当前认识

- 一表通系统当前进入的是基础主数据层（机构信息、机构关系、员工），重点不是业务口径，而是字段定义、编码规范、格式约束和版本变更。
- 与 1104 这类报表型系统相比，一表通系统更适合先按“系统页 + 报表页 + 数据字典型报表实体页 + 原文页”的方式组织。
- 1.1 机构信息与 1.2 机构关系组合后，可以表达“机构属性 + 机构层级关系”的基础模型，可作为后续同类基础数据表的范式。
- 4.1-4.4 共同补齐了一表通的账务核算层，形成“科目主数据 + 总账快照 + 分户账主数据 + 分户账变动快照”的最小骨架。
- 5.1、5.3-5.6 开始补齐产品业务层，形成“通用产品主数据 + 专项产品表（债券/理财/保险/卡）”的结构。
- 6.1-6.7 开始补齐协议与授信链路层，形成“存款/贷款主协议 + 项目贷/互联网贷/银团贷专项协议 + 受托支付/展期执行层”的结构。
- 6.8-6.10 继续把担保、信用卡、贸易融资纳入协议层，补齐担保桥、卡协议和贸易融资专项协议三个关键分支。
- 后续如果继续导入一表通其他表，优先在 [[04-综合/一表通系统-报表关系|一表通系统-报表关系]] 中补报表行，再决定是否需要抽主题页。
