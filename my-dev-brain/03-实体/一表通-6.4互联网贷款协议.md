# 一表通-6.4互联网贷款协议

## 简介

采集报送机构与合作方签订的互联网贷款合作协议信息。互联网贷款的认定参照《商业银行互联网贷款管理暂行办法》。

## 关键字段

| 数据项编码 | 数据项名称 | 说明 |
|------------|------------|------|
| F040001 | 机构ID | PK，强制性校验，一表通拼接金融许可证号前11位 |
| F040002 | 协议ID | PK，强制性校验 |
| F040003 | 借据ID | PK，强制性校验，已签合同未发放贷款可为空 |
| F040004 | 合作协议ID | 合作模式必填，独立模式可为空 |
| F040005 | 业务模式 | PK，强制性校验，4 种码值 |
| F040006 | 合作方负有担保责任的金额 | - |
| F040007 | 合作方出资发放贷款金额 | 新增字段，强制性校验 |
| F040008 | 本机构出资发放贷款金额 | 新增字段，强制性校验 |
| F040009 | 客户数据授权书编号 | 一表通新增字段，强制性校验 |
| F040010 | 授权生效日期 | 强制性校验，日期格式需转换 |
| F040011 | 授权终止日期 | 强制性校验，日期格式需转换 |
| F040012 | 提供部分风险评价服务合作机构名称 | 新增字段 |
| F040013 | 提供担保增信合作机构名称 | 新增字段 |
| F040017 | 合作方协议责任金额 | 2.1 版新增，独立模式填 0 |
| F040018 | 申请人联系电话 | 2.1 版新增，不允许为空 |
| F040015 | 备注 | 2.1 版定义格式长度 |
| F040016 | 采集日期 | PK，强制性校验，日期格式需转换 |

## 业务模式码值

| 码值 | 含义 |
|------|------|
| 01 | 本机构主要作为资金提供方共同出资发放贷款 |
| 02 | 本机构主要作为信息提供方共同出资发放贷款 |
| 03 | 银行单独出资合作发放互联网贷款（100% 出资方） |
| 04 | 本机构独立开展互联网贷款业务（无合作方） |

> 一表通码值比 EAST 更细，需做码值映射。

## 填报规则

- 报送机构与合作方签订的互联网贷款合作协议
- 状态已终结借据报送至当年末
- 互联网贷款认定参照《商业银行互联网贷款管理暂行办法》
- 监管反馈（20231107）：为实现一表通转换 EAST，建议在当月仍继续报送，次月不报

## 版本差异

- **2.1 版新增**：F040017 合作方协议责任金额、F040018 申请人联系电话
- **2.1 版延长**：F040002 协议ID、F040003 借据ID 数据格式长度
- **2.1 版补充说明**：F040003 借据ID"对于存在已签合同未发放贷款的允许为空"
- **2.1 版定义**：F040015 备注数据格式长度

## 数据转换要点

- 日期格式：一表通为 `YYYY-MM-DD`（10 位），EAST 为 `YYYYMMDD`（8 位），需转换
- 业务模式：一表通 4 码值 vs EAST 2 码值，需映射
- 机构ID：一表通需拼接金融许可证号前 11 位
