# 一表通-2.5-个人客户基本情况

## 简介

`2.5个人客户基本情况` 是一表通系统中个人客户主数据的核心报表，用于采集上报机构的所有个人客户基本信息，含以个人名义注册的个体工商户和小微企业主。个人客户范围应包含所有业务，如存款、贷款、中间业务等，按客户ID去重后报送。

## 当前结论

- 这是一表通个人客户主数据表，采集范围覆盖上报机构的所有个人客户基本信息，含以个人名义注册的个体工商户和小微企业主，按客户ID去重后报送。
- 与 `1.1机构信息` 的关系：`2.5` 通过 `机构ID`（`B050002`）与 `1.1` 关联，实现"机构-个人客户"的挂接，机构ID编码规则与 `1.1` 一致（前11位金融许可证号 + 后13位内部机构号）。
- 该表共有 36 个字段，全部为 `FIELD` 类型，无 `INDICATOR`，是典型的主数据定义表。
- 引用 8 张码表：`YBT_GRKHLX`（个人客户类型，5码值）、`YBT_ZJLX`（证件类型，103码值）、`YBT_XB`（性别，2码值）、`YBT_XL`（学历，39码值）、`YBT_SF`（是否类，7次复用）、`YBT_DWXZ`（单位性质，7码值）、`YBT_XZQH`（行政区划）、`YBT_GJDQ`（国家地区）。
- 客户识别体系至少同时包含：客户ID（`B050001`）、机构ID（`B050002`）、个人客户名称（`B050003`）、证件信息（身份证+护照+其他证件类型+其他证件号码）四重维度。
- 个人客户类型（`B050004`）是核心区分字段，5类涵盖：01-普通个人客户、02-个体工商户、03-小微企业主、04-境外客户、00-其他。
- `2.5` 与 `2.4` 的关系：`2.4` 采集个体工商户和小微企业主（经营属性视角），`2.5` 采集所有个人客户（个人属性视角），报送范围存在交叉但采集维度不同。`2.4` 侧重经营信息、财务信息和经营者信息，`2.5` 侧重个人身份、婚姻、教育、职业、收入等个人属性。
- 是否类标识字段共 7 个（`YBT_SF` 复用），覆盖：已婚标识、本行员工标识、上本行黑名单标识、居民标识、农户及新型农业经营主体标识、已脱贫人口标识、边缘易致贫人口标识，形成个人客户的风险与社会属性多维画像。
- 证件信息采用"身份证+护照+其他证件"的三元结构，其中 `B050005`（客户身份证）和 `B050006`（客户护照号）为独立字段，`B050007`（其他证件类型）+`B050008`（其他证件号码）为扩展字段，配合 `YBT_ZJLX` 码表支持 103 种证件类型。
- 收入字段包含个人年收入（`B050022`）和家庭收入（`B050023`）两层，均折算人民币报送，支持个人与家庭两个维度的经济能力评估。

## 重点口径

- 客户ID：长度不超过60位数字与字母组合，不足部分不用补齐。银行机构需保证机构内部针对同一个户有且仅有一个客户ID。
- 机构ID：与 `1.1机构信息` 中的机构ID生成规则一致，前11位跟随金融许可证号，后13位使用内部机构号。
- 个人客户名称：按有效身份证件上排序在第一个的姓名记录，其他语种姓名可记录在其他字段中。
- 个人客户类型（5类）：01-普通个人客户 / 02-个体工商户 / 03-小微企业主 / 04-境外客户 / 00-其他。
- 证件三元结构：身份证（`B050005`）+ 护照号（`B050006`）+ 其他证件类型（`B050007`）+ 其他证件号码（`B050008`），其中其他证件需与 `YBT_ZJLX` 码表配对使用。
- 学历码表（39类）：覆盖研究生教育到小学教育的各层次毕业/结业/肄业状态，以及"其他"选项。
- 已婚标识：0-否 / 1-是。
- 电话格式：境内手机11位无需国家区号；境外手机需加国家区号；境内固话含国内长途区号；境外固话需加国家区号。
- 单位性质（7类）：01-国有企业 / 02-民营企业 / 03-政府机关 / 04-事业单位 / 05-社会团体 / 06-境外机构 / 00-其他。
- 个人年收入和家庭收入：均折算人民币报送，小微个人客户填报经营收入指标，学生等无收入客户可填0。
- 行政区划：按居住地→工作地→户籍地优先级顺序填报。
- 国家地区：GB/T 2659-2000 3位代码，如CHN。
- 首次建立信贷关系年月：默认值 9999-12，不涉及可为空。
- 黑名单日期：默认值 9999-12-31。
- 农户定义：长期（一年以上）居住在乡镇行政管理区域内的住户，含城关镇所辖行政村范围内的住户和户口不在本地但在本地居住一年以上的住户。
