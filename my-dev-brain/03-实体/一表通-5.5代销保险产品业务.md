# 一表通-5.5代销保险产品业务

## 简介

一表通系统报表，采集填报机构所有代销保险产品业务的个性信息。与 [[03-实体/一表通-5.1产品业务基本信息|5.1产品业务基本信息]]（共性信息表）和 [[03-实体/一表通-5.3债券产品业务|5.3债券产品业务]]、[[03-实体/一表通-5.4理财产品业务|5.4理财产品业务]] 同属产品/业务基础信息数据域，但专注保险代销场景。

- 所属系统：一表通
- 报表编码：5.5
- 报表名称：代销保险产品业务
- 数据域：产品/业务基础信息
- 当前原文依据：[[01-资料库/一表通系统/2026-04-27-5.5代销保险产品业务-填报说明-原文]]

## 当前结论

- 本报表为 1.0 版整表新增，无历史版本。
- 采集范围为填报机构所有代销保险产品业务，覆盖主险与附加险的组合关系。
- 主险与附加险直接存在对应关系的，每条记录填报一个主险及其中一个附加险；不存在对应关系的，每个险种填报一条记录。
- 核心业务目的是穿透记录银行代销保险产品的逐笔明细，区分主险/附加险层级。

## 重点口径

### 主险/附加险填报规则

- 主险与附加险直接存在对应关系：每条记录填报一个主险及其中一个附加险（一对多场景下选一条附加险）。
- 主险与附加险不存在对应关系：每个险种各自填报一条记录。

### 险种子类型代码

| 代码 | 险种 |
|------|------|
| 1001 | 普通寿险 |
| 1002 | 分红寿险 |
| 1003 | 投资连结保险 |
| 1004 | 万能保险 |
| 1005 | 年金保险 |
| 1006 | 健康保险 |
| 1007 | 意外伤害保险 |
| 2001 | 企业财产保险 |
| 2002 | 责任保险 |
| 2003 | 信用保险 |
| 2004 | 保证保险 |
| 2005 | 其他保险 |

- 代码格式为 4 位数字，如不适用可不填。

### 产品ID生成规则

- 长度不超过 32 位的数字与字母组合。
- 理财产品：前 14/15 位使用产品登记编码，后不足部分不用补齐。
- 其他产品：使用内部产品编号，不足部分不用补齐。

### 机构ID生成规则

- 长度不超过 24 位的数字与字母组合。
- 前 11 位与金融许可证号前 11 位相同，后 13 位使用内部机构号，不足部分不用补齐。
- 理财登记中心统一使用金融许可证。
- 发行地涉及多个国家时，填写共同上级行机构ID，支持多地区填报。

### 保险公司名称

- 以金融机构许可证登记名称为准；没有金融许可证的，填写法人全称+内部规范名称。

## 重点规则

- 复合主键：E050001（产品ID）+ E050002（机构ID）+ E050007（附加险产品编号）+ E050010（采集日期）。
- 附加险产品编号和附加险名称在无附加险场景下允许为空。
- 备注字段在 2.1 版本中由 `anc` 扩展为 `anc..600`，并删除了"不同备注事项用英文半角分号隔开"的要求。
