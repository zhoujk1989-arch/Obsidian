## 关键报表关系

| 关系 | 说明 |
|------|------|
| 1.1 ↔ 系统基础主数据 | 1.1 是当前一表通系统进入知识库的第一张基础主数据表，承接机构级核心标识和属性信息 |
| 1.2 ↔ 系统基础主数据 | 1.2 是机构层级关系表，承接机构与上级管理机构的隶属关系 |
| 1.3 ↔ 系统基础主数据 | 1.3 是人员主数据表，通过机构ID与机构主数据关联，并补全人员生命周期 |
| 1.1 ↔ 版本演进 | 本表已体现 1.0、2.0、2.1 三个版本阶段，可用于跟踪字段新增路径 |
| 1.1 ↔ 1.2 | 1.1 管机构属性，1.2 管机构层级；通常联表使用 |
| 1.1 ↔ 1.3 | 1.3 通过机构ID关联 1.1；通常联表使用 |
| 1.4 ↔ 1.1 / 1.3 | 1.4 通过机构ID与机构主数据关联，并与员工表里的岗位字段形成补充关系 |
| 1.5 ↔ 1.1 | 1.5 通过机构ID关联机构主数据，形成“机构-渠道设备”归属关系 |
| 1.6 ↔ 1.1 | 1.6 通过机构ID关联机构主数据，形成“机构-股东/关联方”关系视图 |

## 当前结论

- 一表通系统当前以“数据字典型报表”为主，报表重点是字段定义、格式约束和版本演进。
- 1.1 机构信息、1.2 机构关系、1.3 员工、1.4 岗位信息、1.5 自助机具、1.6 股东及关联方信息共同构成当前主数据最小集（机构属性 + 层级 + 人员 + 岗位 + 渠道设备 + 股权关系）。
- 后续新增一表通报表时，应先补本页的报表行和简单描述，再下沉到具体报表实体页。
