## 关键报表关系

| 关系 | 说明 |
|------|------|
| 1.1 ↔ 系统基础主数据 | 1.1 是当前一表通系统进入知识库的第一张基础主数据表，承接机构级核心标识和属性信息 |
| 1.2 ↔ 系统基础主数据 | 1.2 是机构层级关系表，承接机构与上级管理机构的隶属关系 |
| 1.3 ↔ 系统基础主数据 | 1.3 是人员主数据表，通过机构ID与机构主数据关联，并补全人员生命周期 |
| 1.1 ↔ 版本演进 | 本表已体现 1.0、2.0、2.1 三个版本阶段，可用于跟踪字段新增路径 |
| 1.1 ↔ 1.2 | 1.1 管机构属性，1.2 管机构层级；通常联表使用 |
| 1.1 ↔ 1.3 | 1.3 通过机构ID关联 1.1；通常联表使用 |
| 1.4 ↔ 1.1 / 1.3 | 1.4 通过机构ID与机构主数据关联，并与员工表里的岗位字段形成补充关系 |
| 1.5 ↔ 1.1 | 1.5 通过机构ID关联机构主数据，形成“机构-渠道设备”归属关系 |
| 1.6 ↔ 1.1 | 1.6 通过机构ID关联机构主数据，形成“机构-股东/关联方”关系视图 |
| 1.7 ↔ 1.3 / 1.4 | 1.7 通过员工ID和岗位编号分别关联员工和岗位主数据，形成“人员-岗位-柜员”映射 |
| 1.7 ↔ 1.5 | 1.7 中“是否实体柜员”与 1.5 自助机具中的“虚拟柜员ID”存在潜在关联，需要后续补规则确认 |
| 2.1 ↔ 1.6 | 2.1 管对公客户主体信息，1.6 管股东与关联方关系；后续可形成“客户主体-股权关系”联合视图 |
| 2.2 ↔ 2.1 | 2.2 通过“母公司客户ID”与 2.1 单一法人客户建立显式关联，形成“法人客户-集团客户”映射 |
| 2.3 ↔ 2.1 / 2.2 | 2.3 与 2.1 同属客户主数据域，但对象是同业机构；后续可与 2.2 一起形成客户分层视图 |
| 2.4 ↔ 2.1 / 2.3 | 2.4 与 2.1、2.3 同属客户主数据域，但对象是个体经营者/小微客户；后续可形成客户分层视图 |

## 当前结论

- 一表通系统当前以“数据字典型报表”为主，报表重点是字段定义、格式约束和版本演进。
- 1.1 机构信息、1.2 机构关系、1.3 员工、1.4 岗位信息、1.5 自助机具、1.6 股东及关联方信息、1.7 柜员、2.1 单一法人基本情况、2.2 集团基本情况、2.3 同业客户基本情况、2.4 个体工商户及小微企业主基本情况共同构成当前主数据最小集（机构属性 + 层级 + 人员 + 岗位 + 渠道设备 + 股权关系 + 柜面角色 + 对公客户 + 集团客户 + 同业客户 + 小微客户）。
- 后续新增一表通报表时，应先补本页的报表行和简单描述，再下沉到具体报表实体页。
