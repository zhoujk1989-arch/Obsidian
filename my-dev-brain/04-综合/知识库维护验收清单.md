# 知识库维护验收清单

本页把 ingest、query、lint 的结束检查固定下来，减少 agent 依赖临时记忆。执行时先按操作类型验收，再记录到 [[05-日志/log|日志]]。

## 通用最低验收

| 检查项 | 通过标准 | 未通过动作 |
|--------|----------|------------|
| 入口读取 | 已读取 [[00-首页/index|首页]]、最近相关日志和任务直接相关页面 | 补读入口和相关页后再操作 |
| 承载页选择 | 新内容落在资料层、知识层、队列层或日志层中的合适位置 | 优先更新已有页面，避免新建近义页 |
| 交叉链接 | 新增或更新页面能回到来源、系统、主题或项目入口 | 补双链或路径链接 |
| 不确定性 | 口径冲突、来源缺失、原文缺项已标注 `待确认` | 写入 [[04-综合/待确认清单|待确认清单]] |
| 日志 | 重要操作已写入 [[05-日志/log|日志]] | 补 type、summary、changed、核心要点 |
| 经验回收 | 本次可复用经验已判断是否需要回写 | 能复用就写入工作台、验收清单、待确认清单或规则文件 |

## Ingest 验收

| 检查项 | 通过标准 |
|--------|----------|
| 类型判断 | 已判断资料属于普通资料、监管原文、数据字典、版本更新、勘误、代码表或项目资料 |
| 去重检索 | 已按标题、报表编码、系统名、版本号、物理表名、关键词查过现有页面 |
| 版本策略 | 同版本去重，新版本新建版本原文页或追加版本差异，未覆盖旧原文正文 |
| 原文/资料页 | 新资料已进入 `01-资料库/`；原文页保持忠实转录，不把结论写进原文主体 |
| 元信息 | 来源、作者、发布时间、获取日期、类型、可更新页面已尽量补齐；缺失项写明 `待确认` |
| 知识回写 | 至少更新 1 个相关主题页、实体页、系统页、报表页或综合页 |
| 去冗余 | 字段全量清单优先保留在原文页；实体页只保留字段分组、关键字段、规则和口径 |
| 明细原文模板 | 监管类明细接口表、数据字典和字段清单原文页优先使用 [[90-模板/监管明细原文模板|监管明细原文模板]]，按报表首页、字段总表、字段填报说明、码值/校验/版本记录分区 |
| 监管路由 | 监管资料已更新系统实体页、监管系统清单、系统报表关系页和报表实体页中的必要入口 |
| 批量剩余 | 批量资料未处理完时，剩余范围已写入 [[00-首页/当前工作台|当前工作台]] |
| 典型查询 | 用 2 到 3 个问题验证能先命中知识页，而不是直接回到原文全文搜索 |

## Query 验收

| 检查项 | 通过标准 |
|--------|----------|
| 问题类型 | 已判断属于事实查找、口径解释、跨表对比、差异核对、项目状态、巡检修复或开放问题 |
| 路由路径 | 监管类问题按 L0 系统页 → L1 系统报表页 → L2 报表/规则页 → L3 原文页读取 |
| 候选集 | 已从系统清单、系统报表关系页、业务描述、场景/关系页和关键词检索形成候选文件清单 |
| 必读文件 | 单表问题已读对应实体页；跨表、场景、口径和开放问题已读完候选集中的必读实体页、规则页、关系页或场景页 |
| 二次复检 | 回答前已用用户关键词、同义词、报表编码或业务场景词在 `04-综合/`、`03-实体/`、`02-主题/` 中复检 |
| 排除说明 | 新增候选已读取或说明排除理由；未读取页面没有被写成事实依据 |
| 停手条件 | 单表事实可在实体页足够时停止；跨表和场景问题必须完成候选集、必读文件和二次复检后再停手 |
| 结论分层 | 回答中区分“已知结论”“依据来源”“推断判断”“待确认” |
| 长期价值 | 本次产生的新结论、新对比或新框架已回写到既有知识页或综合页 |
| 待确认项 | 无法确认的问题已进入 [[04-综合/待确认清单|待确认清单]] |
| 回归样例 | 可能复现的问题已判断是否加入查询回归样例或 FAQ/规则页 |
| 日志 | 有沉淀价值的 query 已记录到日志 |

## Lint 验收

### 巡检分级

| 层级 | 检查范围 | 输出要求 |
|------|----------|----------|
| L1 结构巡检 | frontmatter、断链、重复链接、异常列表前缀、根目录临时文件、超长行、命名不一致 | 能自动修的直接修，不能判断的记待确认 |
| L2 路由巡检 | 首页、工作台、系统清单、系统报表页、资料目录之间的入口一致性 | 修入口、计数和链接，不重复堆报表明细 |
| L3 知识巡检 | 重复页面、冲突结论、待确认积压、高频概念未建页、规则页缺失 | 只处理有证据的问题，口径判断不足则进待确认 |

### 系统页瘦身验收

| 检查项 | 通过标准 |
|--------|----------|
| 系统实体页 | 只保留系统定位、覆盖范围、关键入口、系统级规则和少量稳定结论 |
| 报表明细 | 已下沉到 `04-综合/系统名-报表关系.md` |
| 原文清单 | 已下沉到 `01-资料库/系统名/README.md` |
| 变化记录 | 只保留系统级变化；逐张报表新增记录进入系统报表页或日志 |
| 模板依据 | 新建或重构系统实体页优先使用 [[90-模板/系统实体模板|系统实体模板]] |

### 系统报表目录页验收

| 检查项 | 通过标准 |
|--------|----------|
| 目录定位 | `04-综合/系统名-报表关系.md` 只维护系统与报表/数据表的包含关系和短业务描述 |
| 必要字段 | 至少包含报表编码/表编号、报表名称、领域/数据域、业务描述、报表实体页、原文入口、状态 |
| 内容边界 | `业务描述` 只写一句检索路由摘要；系统概要回系统实体页；报送日期、频度、字段、口径回报表实体页；表间校验和完整业务场景进入关系页、场景页或规则页 |
| 原文入口 | 字段全量清单和原文依据回到 `01-资料库/系统名/README.md` 或具体原文页 |
| 模板依据 | 新建或重构系统报表页优先使用 [[90-模板/监管报表页模板|监管报表页模板]] |

### 可自动检查和修复

| 检查项 | 修复原则 |
|--------|----------|
| frontmatter | `02-主题/`、`03-实体/` 中应有合法 YAML frontmatter |
| 断链 | 优先更新到现存页面；无法判断时记录待确认，不删除来源 |
| 重复链接/重复行 | 保留信息更完整、命名更稳定的一条 |
| 计数不一致 | 先核实实体页、报表关系页、系统清单三者口径，再更新计数 |
| 非法列表前缀 | 清理 `|-`、`||-` 等误生成前缀，恢复为普通列表或表格 |
| 根目录临时文件 | 能归档的归档到对应日志/巡检页；不能判断用途时先保留并记录 |
| 超长行 | 优先拆为短段、列表或表格；`系统名-报表关系.md` 中的非目录内容迁到系统实体页、报表实体页、关系页或场景页 |

### 必须待确认

| 情况 | 处理方式 |
|------|----------|
| 原文缺项或编号异常 | 不自行补写定义；在知识页标注依据并进入待确认 |
| 多版本差异 | 新建版本原文页或追加版本差异，不直接覆盖旧原文 |
| 口径冲突 | 明确冲突来源与日期，保留旧结论并标注依据 |
| 跨表关系推断 | 只有原文或制度明确支持时才写成结论，否则写成推断或待确认 |

### 巡检报告最低结构

- 范围：本次检查了哪些目录、系统或页面。
- 发现：按结构、路由、知识三类列出问题。
- 已修复：列出实际修改页面和修复动作。
- 待确认：列出证据不足或不宜自动判断的问题。
- 后续优先级：列出下一轮最值得处理的 3 到 5 项。

## Fix 验收

| 检查项 | 通过标准 |
|--------|----------|
| 范围清晰 | 已说明修复对象属于格式、链接、计数、索引、命名、重复、归档等结构问题 |
| 原文安全 | 未改写原文主体；涉及原文缺项、多版本差异或口径判断时已转入待确认 |
| 最小改动 | 只修复当前问题，不顺手重构无关页面 |
| 复用经验 | 若修复暴露可重复检查项，已补入本页或当前工作台 |
| 日志 | 已以 `fix` 类型记录到 [[05-日志/log|日志]] |

## 推荐巡检命令

这些命令只作为低成本检查入口，结果需要结合页面语义判断。

```bash
rg -n '^\|+-|^\|{2,}-' 00-首页 03-实体 04-综合
rg -n '待确认|待补充|待建|TODO' 02-主题 03-实体 04-综合 06-项目
rg --files -g '*.md' | awk -F/ '{count[$1]++} END {for (d in count) print count[d], d}' | sort -nr
wc -l 00-首页/index.md 04-综合/监管系统-报表清单.md 04-综合/*系统-报表关系.md
```

## 查询回归样例

每次大规模入库或巡检后，至少抽 2 到 3 个问题检查路由是否有效：

- `G14_V 大额风险暴露统计表当前原文依据是什么？`
- `一表通 6.10 贸易融资协议应该先看哪个实体页和原文页？`
- `S67 房地产贷款合计与 G01_Ⅶ 房地产业是否已有明确表间校验？`
- `银行发放一笔个人住房按揭贷款，应在一表通和 1104 分别关注哪些报表和指标？`

## 自我改进验收

当本次工作暴露出新的失败模式、检索捷径或高频问题时，按以下表格处理：

| 经验类型 | 承载位置 | 示例 |
|----------|----------|------|
| 临时待办或下一步 | [[00-首页/当前工作台|当前工作台]] | 某系统页需要瘦身、某报表计数待核实 |
| 可重复检查项 | 本页 | 新增断链检查、计数检查、异常列表前缀检查 |
| 查询回归样例 | 本页“查询回归样例” | 高频问题可以固定为回归样例 |
| 稳定跨任务规则 | [[AGENTS|AGENTS.md]]，并同步 `CLAUDE.md` | 多次出现的流程性误差 |
| 业务口径问题 | [[04-综合/待确认清单|待确认清单]] 或对应知识页 | 原文缺项、跨表关系缺证据 |

只有当经验能转成具体动作时才写入规则文件；如果只是一次性提醒，优先写入工作台或日志。

## 结束判断

一次维护完成时，至少满足以下 4 项中的 3 项：

1. 受影响页面已更新。
2. 关键交叉链接已补齐。
3. 高价值结论已进入知识层或队列层。
4. 日志已记录本次操作。
5. 对应操作类型的验收项没有明显缺口。
6. 可复用经验已完成回收判断。
