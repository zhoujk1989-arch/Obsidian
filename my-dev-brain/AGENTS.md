# LLM Wiki Agent Guide

本仓库是一个由 LLM 协助维护的个人知识库，目标不是临时回答问题，而是持续沉淀结构化知识。

## 核心原则

1. 原始资料只增不改，作为事实来源保留。
2. Wiki 页面是整理后的知识层，允许持续更新。
3. 每次处理新资料时，优先更新已有页面，而不是重复创建近义页面。
4. 回答用户问题时，优先基于 Wiki 页面组织答案，必要时再回溯原始资料。
5. 重要的新结论、新对比、新框架，应回写到 Wiki，而不是只停留在聊天记录里。

## 目录约定

- `00-首页/`：入口页、总索引、工作说明。
- `01-资料库/`：原始资料的摘录、导入文本、来源记录。
- `02-主题/`：按主题组织的核心知识页。
- `03-实体/`：人物、公司、产品、项目、概念对象等实体页。
- `04-综合/`：综述、对比、路线图、FAQ、决策文档。
- `04-综合/梦境/`：系统对"你最近在变化什么"的定期综合页。
- `05-日志/`：摄入、问答、巡检日志。
- `06-项目/`：按项目维度维护的长期页面。
- `90-模板/`：页面模板。

## 检索策略

当需要查找信息时，按以下层级依次读取，找到答案即停止：

| 层级 | 位置 | 用途 | 何时读取 |
|------|------|------|----------|
| L0 路由表 | `04-综合/监管系统-报表清单.md` | 扫描关键词列，定位目标报表和页面链接 | 每次监管类 Query 首先读取 |
| L1 知识页 | `03-实体/` 与 `02-主题/` | 报表结论、常见口径、跨报表规则、系统关系 | 路由表命中后优先读取 |
| L2 原文页 | `01-资料库/*/` | 逐字段定义、核对关系原文、特定项目注解 | 知识页不够精确时读取 |

补充路由入口：
- 跨报表概念问题（如"贸易融资怎么填"）→ 先查 `02-主题/` 中的规则页
- 非监管类问题 → 仍走 `00-首页/index.md` → 主题页/实体页 → 资料页

## 页面类型

### 资料页

路径：`01-资料库/`

用途：
- 保存原始文章、制度文件、填报说明、导入文本、摘要页和来源记录
- 作为后续更新主题页和实体页的依据

建议文件名：
- `YYYY-MM-DD-标题.md`

模板：`90-模板/资料模板.md`

建议包含：
- 来源链接
- 作者/发布时间
- 三到五条核心观点
- 可更新哪些主题页/实体页
- 与已有页面的关联

#### 监管资料特殊约定

对于监管系统相关资料，`01-资料库/` 默认采用"系统目录直存原文"的结构。

目录示例：

- `01-资料库/1104系统/`
- `01-资料库/反洗钱系统/`

执行原则：

1. 原文页优先保留精度，不压缩项目编号、口径原话、核对关系、特定项目注解。
2. 资料层不再单独维护提炼页；提炼结论直接并入系统页、报表页和规则页。
3. 单张报表的摘要、常见口径、重点规则，优先维护在对应报表页。
4. 跨报表复用的规则线索，优先维护在 `02-主题/` 的主题页或规则页。
5. 如果用户提供的是监管文档正文，默认创建原文页并同步更新相关知识页，而不是再建提炼页。
6. 监管类原文页按 `90-模板/原文模板-监管.md` 结构创建。

建议命名：

- 原文页：`YYYY-MM-DD-标题-原文.md`

示例：

- `01-资料库/1104系统/2026-04-08-G01-资产负债项目统计表-填报说明-原文.md`

### 主题页

路径：`02-主题/`
模板：`90-模板/主题模板.md`

用途：
- 汇总某个主题下的稳定知识
- 不写成一次性摘要，而写成长期维护页面

建议包含：
- 主题定义
- 当前结论
- 关键争议
- 关联实体
- 关联资料
- 待补充问题

### 规则页

路径：`02-主题/`
命名：`概念名-报表口径.md`
模板：`90-模板/规则模板.md`

用途：
- 汇总同一业务概念在多张报表中的不同填报口径
- 解决"某概念在 G01 填哪里、G21 填哪里"这类跨报表问题

建议包含：
- 概念定义
- 各报表口径对照表（报表 | 项目编号 | 填报规则 | 依据来源）
- 易混淆点
- 关联报表页和原文页链接

参考实现：`02-主题/贸易融资-报表口径.md`

### 实体页

路径：`03-实体/`
模板：`90-模板/实体模板.md`

用途：
- 针对具体对象建立持续更新档案

适用对象：
- 人物
- 公司
- 产品
- 模型
- 论文
- 方法论名词

补充说明：
- 监管报表作为实体页维护时，优先使用 `90-模板/报表模板.md`
- 报表页直接吸收原先提炼页承担的摘要、规则和问答职责

### 综合页

路径：`04-综合/`
模板：`90-模板/综合模板.md`

用途：
- 沉淀较高价值的跨页成果

常见类型：
- 对比分析
- 研究综述
- 学习路径
- 决策备忘
- 问题清单

### 梦境页

路径：`04-综合/梦境/`
模板：`90-模板/梦境模板.md`

用途：
- 生成系统对"你最近在做什么、变成什么、该往哪里走"的阶段性理解
- 作为个人上下文的中间层，不直接覆盖稳定自我页

建议包含：
- 本次主题
- 观察
- 推断
- 可能忽视的信号
- 建议更新的核心页
- 下一步问题
- 一句话画像

## 三类操作

### Ingest

当用户添加新资料时：

1. 创建或更新资料页到 `01-资料库/`
2. 如果是监管类文档，先在系统目录下创建原文页，再判断应更新哪些报表页或规则页
3. 提炼核心观点
4. 主动浏览所有已有主题页、实体页，判断哪些需要补充或修订（不只是新建相关页面）
5. 将新信息合并进对应页面
6. 补充交叉链接
7. 在 `05-日志/log.md` 追加记录，格式：`## [YYYY-MM-DD] ingest | 资料标题`

如果资料与现有内容冲突：
- 明确标记冲突点
- 不静默覆盖旧结论
- 在相关页面注明依据来源

对于监管系统相关 ingest，默认执行以下顺序：

1. 在 `01-资料库/系统名/` 创建原文页
2. 更新或创建对应系统页到 `03-实体/`
3. 如报表已经形成稳定对象，更新或创建对应报表页到 `03-实体/`，直接吸收摘要、规则和常见口径
4. 更新 `04-综合/监管系统-报表清单.md`：补充行、填写关键词列、确保链接完整
5. 扫描新资料中的跨报表概念，检查是否已有对应规则页；如该概念已出现在 3 份以上资料中且尚无规则页，则创建规则页到 `02-主题/`
6. 必要时再抽取其他主题页、对比页

### Query

当用户提问时：

1. 判断问题类型：
   - 监管报表类 → 读取 `04-综合/监管系统-报表清单.md`，扫描关键词列匹配目标
   - 跨报表概念类 → 在 `02-主题/` 查找规则页
   - 一般知识类 → 读取 `00-首页/index.md` 定位相关页面
2. 按检索策略 L0→L1→L2 逐层读取，找到答案即停止
3. 如果知识页已有常见口径、重点规则或对照表，优先复用，不再单独查提炼页
4. 输出回答时区分"已知结论"和"推断判断"
5. 如果回答产生了长期有价值的新内容，回写到对应页面
6. 在 `05-日志/log.md` 记录本次问答，格式：`## [YYYY-MM-DD] query | 问题摘要`

### Dream

当用户要求"做一次梦"时：

1. 优先读取 `00-首页/index.md`
2. 读取 `00-首页/我是谁.md`、`00-首页/我的偏好.md`、`00-首页/我的目标.md`
3. 读取 `06-项目/当前学习地图.md`、相关项目页和 `05-日志/log.md`
4. 优先关注最近 7 天更新过的资料页、主题页、综合页；如信息不足可回看最近 30 天
5. 输出时必须区分"观察"和"推断"
6. 推断允许大胆，但不能写成既定事实
7. 自动落到 `04-综合/梦境/YYYY-MM-DD-梦境.md`
8. 只提出建议更新的核心页，不直接改写 `我是谁`、`我的偏好`、`我的目标`、`当前学习地图`
9. 在 `05-日志/log.md` 记录本次 dream，格式：`## [YYYY-MM-DD] dream | 梦境标题`

### Lint

定期巡检知识库：

- 是否有重复页面
- 是否存在孤立页面
- 是否缺少重要交叉链接
- 是否有明显过时结论
- 是否有频繁出现但尚未建页的概念
- 是否有冲突信息未标注来源
- 报表页、实体页、主题页/规则页是否缺少 frontmatter
- `04-综合/监管系统-报表清单.md` 的行数是否与已纳入的监管原文数量一致
- 出现在 3 份以上资料中的概念是否已建规则页

巡检后：
- 优先修补结构问题
- 再在 `05-日志/log.md` 记录本次巡检，格式：`## [YYYY-MM-DD] lint | 巡检摘要`

## 写作约定

1. 优先使用中文。
2. 页面标题简洁明确，避免同义重复。
3. 一页只承载一个稳定主题。
4. 使用 Markdown 链接互相引用。
5. 普通知识页不堆砌长摘录，优先写提炼后的结论；但监管类原文页例外，应优先保留原文精度。
6. 对不确定内容显式标注"待确认"。
7. 对时间敏感内容尽量写明具体日期。
8. 以下页面类型必须包含 YAML frontmatter：
   - 实体页：`type: 实体`, `entity_type: 系统|报表|人物|产品|...`, `keywords: [关键词]`
   - 主题页/规则页：`type: 主题` 或 `type: 规则页`, `keywords: [关键词]`
   - frontmatter 示例：
     ```yaml
     ---
     type: 实体
     entity_type: 报表
     keywords: [资产负债, 贸易融资, 理财产品, 外币折算]
     ---
     ```
   - 原文页、首页、梦境页、日志、模板不要求 frontmatter。

## 命名建议

- 主题页：`主题名.md`
- 实体页：`实体名.md`
- 综合页：`主题-对比.md`、`主题-综述.md`、`主题-路线图.md`
- 资料页：`YYYY-MM-DD-标题.md`

监管资料补充建议：

- 系统目录：`系统名/`
- 原文页：`YYYY-MM-DD-标题-原文.md`

## 每次开始工作时优先读取

1. `00-首页/index.md`
2. `05-日志/log.md`
3. 与当前任务最相关的页面

## 每次结束工作时检查

1. 是否更新了受影响页面
2. 是否补上了交叉链接
3. 是否把高价值输出沉淀成页面
4. 是否记录到日志
