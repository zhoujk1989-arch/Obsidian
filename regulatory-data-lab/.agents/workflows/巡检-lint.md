# 巡检 / Lint 工作流

当用户要求巡检、检查、lint、死链、一致性或状态核对时，执行本流程。

## 检查范围

- `AGENTS.md`、`.agents/`、`索引.md`、`日志.md`
- `报表业务口径/`
- `概念/`
- `数据表/`
- `血缘/`
- `sql/`
- `模板/`

## 核心检查

- 旧目录残留：不得出现已废弃的多层知识目录、历史材料目录、旧 SQL 工作区或旧 SQL 子目录。
- 死链：不得保留指向已删除来源对象的 wikilink。
- Frontmatter：知识页必须具备仓库要求的基础 frontmatter。
- 模块边界：报表页讲业务口径，数据表页讲字段和码值，血缘页讲来源去向，概念页讲复用规则。
- 系统边界：除 `概念/` 外，报表、数据表、血缘、SQL 均按系统目录分层。
- 日志：实质修复后必须追加 `日志.md`。

## 处理原则

- 能确定的结构问题直接修。
- 不能确认的业务问题写入 `Open Questions` 或日志。
- 不制造不存在页面的占位 wikilink。
- 修复后重新运行检查并记录结果。
