# SQL/存储过程开发工作流

当用户要求写 SQL、生成 SQL、补 SQL、写存储过程、生成视图、加工脚本、调度脚本或校验 SQL 时，执行本流程。

必须优先使用 `.agents/skills/sql-dev/` 中的 SQL 开发 skill；涉及血缘页、字段级来源、上下游影响或回链检查时，同时使用 `.agents/skills/regulatory-data-lab-lineage/`。同时遵守 `AGENTS.md` 的仓库级规则。本文件负责仓库内知识回写和交付纪律。

## 需求抽取

先澄清或自行抽取以下要素：

- 目标产物：查询 SQL、`insert select`、存储过程、视图、临时表加工、校验 SQL 或调度片段。
- 目标系统与目标表：必须落在明确系统目录下；跨系统时选择主系统目录并说明涉及系统。
- 业务目标：统计对象、数据粒度、时间口径、纳入规则、排除规则、输出字段、刷新方式。
- 运行约束：数据库方言、跑批日期参数、是否先删后插、是否幂等、是否需要增量处理、性能限制。

## 知识读取顺序

1. 读 `索引.md`，定位系统。
2. 读对应 `概念/概念-系统-<系统名>.md`。
3. 打开相关数据表页、报表业务口径页和必要血缘页。
4. 如果需求依赖监管报送口径、字段是否必报、码值含义或制度解释，按 `.agents/rules/外部来源规则.md` 检索 `regulatory-knowledge-vault`。

不能只凭字段名猜测业务口径。

## SQL 落地

SQL 文件默认写入 `sql/<系统名>/`。

推荐命名：

- 查询脚本：`SQL_<需求名>.sql`
- 存储过程：`PROC_<需求名>.sql`
- 视图定义：`VIEW_<需求名>.sql`
- 实现说明：`说明-<需求名>.md`

## SQL 文件结构

SQL 文件应尽量使用 `模板/SQL开发模板.sql`，至少包含：

- 文件头：业务目标、依赖知识页、源表、目标表、参数、运行方式、未确认点。
- 参数区：跑批日期、机构范围、重跑范围等。
- 清理区：如需要先删后插，删除条件必须与插入范围一致。
- 主加工区：默认使用直接 `select` + `left join` 组织；只有多阶段汇总、窗口去重、复杂复用或可读性明显改善时才使用 CTE。
- 落地区：目标字段顺序必须与目标表或输出清单一致。
- 校验区：行数、主键重复、必填为空、码值越界、金额或余额方向等检查。

## SQL 编写规则

- 业务口径优先于技术写法；SQL 注释必须说明业务含义。
- 默认不使用 `select *`。
- 目标字段顺序必须与目标表 DDL、数据表页字段清单或用户指定输出顺序一致。
- 常量字段、码值映射、条件判断、拼接字段、窗口函数和聚合字段都必须有注释说明依据。
- 金额、余额、日期、机构、客户、账号、合同号等关键字段必须明确来源表和来源字段。
- 关联条件必须写清业务键，不能只凭同名字段关联。
- 过滤条件必须能映射到业务纳入/排除规则；如果只是技术限制，要在注释中说明。
- 对日期口径要区分交易日、自然日、数据日期、会计日期、跑批日期和报送期。
- 删除重跑脚本的 `delete` 范围必须与 `insert` 范围一致。
- 数据库方言不明确时，优先写 ANSI SQL 风格，并在文件头标注“数据库方言待确认”。

## 验证 SQL

生成 SQL 后必须同时给出验证 SQL 或验证清单，至少覆盖：

- 目标行数是否符合业务范围
- 目标主键或唯一性是否重复
- 必填字段是否为空
- 代码类字段是否只出现允许码值
- 金额、余额、日期字段是否存在明显异常
- 与上游来源的抽样回溯是否能对上

## 知识页同步

开发完成后不能只交付 SQL，必须同步维护相关知识页。除非用户明确说“只要 SQL，不更新知识库”，否则未完成本节即视为 SQL 开发未完成。

- 报表业务口径页：如果 SQL 对应报表、接口、结果表或固定业务输出，写清整表口径、字段/指标口径、纳入排除规则、时间口径和代码类字段规则。
- 数据表页：只维护字段主档、码值、表内指标、证据摘要和引用入口；不写字段落地状态、SQL 产出状态、直接上游或直接下游对象。
- 血缘页：至少覆盖本次 SQL 的表级链路、上下游对象、目标输出字段的字段级来源、字段落地状态和证据状态。
- 血缘页：直接上下游对象、Nodes、表级 Edge List、字段级 Edge List、关键过滤与依赖条件中的内部对象和 SQL 文件，已存在时必须使用 Obsidian wikilink；Markdown 表格内 SQL 别名链接必须转义竖线为 `\|`。
- 概念页/系统页：如果产生新的可复用加工规则、系统边界、通用码值解释或日期口径，应更新共享知识页。
- 系统页：补齐具体对象入口、表族摘要或当前问题，不把具体对象平铺进根索引。
- `日志.md`：追加 `sql-dev` 或 `development` 记录。

## 最低交付标准

- 一份位于 `sql/<系统名>/` 的 SQL 或存储过程文件。
- 必要时一份同目录校验 SQL。
- 文件头列明业务目标、依赖知识页、源表、目标表、参数、运行方式和未确认点。
- 关键字段映射、过滤条件、码值转换和聚合逻辑均有依据说明。
- 已同步维护相关报表业务口径页、数据表页和血缘页；如果确无对应对象，必须在交付说明中写明原因。
- 已按血缘维护规则补齐：血缘页完整记录表级边、字段级边、上下游对象、字段落地状态和证据状态；数据表页只保留对应血缘页引用入口。
- 已按血缘链接约束检查：血缘页边表对象和证据链接可点击，表格内 SQL wikilink 未因未转义竖线导致列数错乱。
- 已更新对应系统页和 `日志.md`。
- 回答中列出依据页面和待确认问题。
