# 报表 SQL Ingest 工作流

当用户要求 ingest 一份监管报表 SQL、存储过程、视图定义、结果表加工脚本或标准接口加工 SQL 时，执行本流程。

## 工作步骤

1. 读取 `索引.md` 和对应系统页，确认系统边界和对象入口。
2. 读取或定位 `sql/<系统名>/` 下的相关 SQL 文件。
3. 必要时先用 `工具/血缘脚手架.py` 做第一轮结构化抽取。脚手架只作为初稿，不能替代业务解释和字段级核对。
4. 搜索相关报表业务口径页、数据表页、血缘页、系统页和概念页，优先更新已有对象。
5. 如需要监管业务来源，使用 `工具/检索外部知识库.py` 或按外部来源检索顺序检索 `regulatory-knowledge-vault`，写入知识页时只记录 Obsidian wikilink。
6. 创建或更新 `报表业务口径/<系统名>/报表-...`，面向业务人员说明报表目的、报送范围、纳入排除规则、时间口径和字段/指标口径摘要。
7. 为涉及的核心物理表和稳定逻辑数据集创建或更新 `数据表/<系统名>/` 下的数据表页，只写字段口径、码值说明、表内指标、证据摘要和引用入口；上下游关系和字段产出状态不写入数据表页。
8. 创建或更新 `血缘/<系统名>/血缘-...`，至少覆盖目标输出字段的字段级来源。
9. 按 `.agents/rules/血缘双向维护规则.md` 维护血缘：血缘页统一记录直接上游、直接下游、表级边、字段级边、字段落地状态和证据状态；数据表页只保留血缘页引用入口。
10. 检查是否产生跨报表复用知识，例如系统边界、通用概念、共享口径、代码映射规则；如有，更新系统页或概念页。
11. 对照既有知识页检查口径冲突、旧结论替代、旧链接失效或系统边界变化。
12. 补齐报表业务口径页、数据表页、血缘页、系统页、概念页之间的摘要级互链。
13. 重新判断触达页面的 `status`。
14. 更新对应系统页中的报表入口、表族入口、血缘入口、SQL 入口或当前覆盖摘要；不要把具体对象平铺进根索引。
15. 向 `日志.md` 追加 ingest 记录。

## 字段与码值要求

如果目标对象是接口表、结果表或监管报送表，所有代码类字段都必须在对应数据表字段口径中直接写明：

- 码值
- 中文含义
- 映射规则
- 特殊值或常量值
- 证据来源

缺少 SQL 来源或监管来源的字段，不得写成已闭环，应进入 `Open Questions`。

## 最低交付标准

一次报表 SQL ingest 完成的最低标准是：

- 一张报表业务口径页
- 一张血缘页
- 所有核心数据表页
- 血缘页已按血缘维护规则补齐上下游对象、表级边、字段级边和字段落地状态
- 核心字段和核心指标口径已经写入数据表页
- 接口表中的代码类字段已写入字段口径
- 相关系统页或通用概念页已经检查，并在需要时更新
- 相关页面的 `status` 已重新判断
- 对应系统页和 `日志.md` 已更新
