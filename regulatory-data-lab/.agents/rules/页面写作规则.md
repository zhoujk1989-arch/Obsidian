# 页面写作规则

本文件规定报表业务口径页、数据表页、血缘页和概念页的写作边界。仓库级对象边界以 `AGENTS.md` 为准。

## 通用写作风格

- 业务解释优先使用简洁中文。
- 英文或 `snake_case` 只用于技术标识。
- 优先使用要点和表格。
- 代码片段只保留证明结论所必需的最短部分。
- 不复制大段监管原文。
- 如果页面中出现技术字段名，应同时给出业务名称或中文含义。
- 不创建指向不存在页面的占位 wikilink；待补对象写普通文本。

## 报表业务口径页

报表业务口径页默认面向业务人员阅读，不要写成 SQL 注释翻译稿或开发说明。

最低要求：

- 说明整张表到底报送哪些业务对象。
- 说明哪些对象纳入、哪些对象排除。
- 说明这是全量报送、增量报送还是截面报送。
- 明细类报表逐字段说明口径。
- 汇总类报表逐指标说明统计对象、过滤条件、聚合方式、分子分母和依赖来源。
- 代码类字段必须写清码值或映射来源。
- 新增报表业务口径页默认按 `模板/报表业务口径模板.md` 落地。

## 数据表页

数据表页是字段口径和表内指标口径的默认承载页。

最低要求：

- 表在监管链路中的角色
- 主键与核心标识
- 关键字段与口径
- 表内指标口径（如有）
- 字段落地状态
- 上下游对象
- 相关报表业务口径页和血缘页
- 必要证据摘要

如果该表是接口表、结果表或监管报送表，代码类字段必须逐字段写清码值含义、映射规则、特殊值和证据来源。

## 血缘页

血缘页默认面向技术人员使用，主要服务于影响分析、反向溯源、字段口径核对和变更评估。

每张血缘页都应包含：

1. 业务链路摘要
2. 节点列表
3. 表级边列表
4. 字段级边列表
5. 条件允许时提供 Mermaid 图
6. 已知缺口、人工假设或未确认点

字段级边列表建议包含：源对象、源字段、目标对象、目标字段、处理逻辑、关系类型、证据。

关系类型建议使用：直接映射、条件映射、拼接派生、聚合派生、窗口派生、常量赋值、码值转换。

## 模板联动

创建或更新以下页面时，应同步检查互链是否完整：

- `模板/血缘模板.md`
- `模板/数据表模板.md`
- `模板/报表业务口径模板.md`
