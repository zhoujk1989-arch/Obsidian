# 页面写作规则

本文件规定报表业务口径页、数据表页、血缘页和来源页的写作边界。仓库级对象边界以 `AGENTS.md` 为准。

## 通用写作风格

- 业务解释优先使用简洁中文。
- 英文或 `snake_case` 只用于技术标识。
- 优先使用要点和表格。
- 代码片段只保留证明结论所必需的最短部分。
- 不复制大段监管原文。
- 如果页面中出现技术字段名，应同时给出业务名称或中文含义。
- 不创建指向不存在页面的占位 wikilink；待补对象写普通文本。

## SQL 翻译为业务语言

对于每一份 SQL，先抽取：

- 报表目的
- 统计粒度
- 维度
- 度量
- 纳入规则
- 排除规则
- 时间逻辑
- 来源数据表
- 关联逻辑
- 转换阶段
- 最终输出字段

然后回答：

- 到底在统计什么
- 统计发生在什么日期或粒度上
- 哪些对象被纳入，哪些被排除
- 哪些来源表是权威来源
- 哪些过滤条件会实质性改变业务口径

不要把 SQL 直接原样搬进正文。先用业务语言解释含义，再把 SQL 表达式作为证据保留。

## 报表业务口径页

报表业务口径页默认面向业务人员阅读，不要写成 SQL 注释翻译稿或开发说明。

### 明细类报表

必须完整覆盖两部分：

1. 整表报送范围
2. 每个报送字段的字段级口径

最低要求：

- 说明整张表到底报送哪些业务对象。
- 说明哪些对象纳入、哪些对象排除。
- 说明这是全量报送、增量报送还是截面报送。
- 逐字段说明口径，不能只写重点字段。
- 每个字段说明字段含义、取值或映射逻辑、来源说明、特殊规则。

明细类字段口径中的“来源说明”应优先使用自然中文句式，例如“来源于监管集市机构表的机构地址字段”。技术表名和字段名可放在“规则说明”中作为证据补充。

### 汇总类报表

必须完整覆盖每个指标的取数规则：

- 指标名称
- 统计对象
- 取数规则
- 过滤条件
- 聚合方式
- 依赖来源
- 派生指标依赖的基础指标
- 分子、分母、去重口径、时点口径

### 通用要求

- 业务解释优先，SQL 细节后置。
- 不允许只停留在“源表 + where 条件 + insert 目标表”的摘要层。
- 代码类字段必须写清码值或映射来源。
- 常量字段、拼接字段、判断字段、窗口函数字段都必须显式说明。
- 新增报表业务口径页默认按 `模板/报表业务口径模板.md` 落地。

## 一表通字段版本判断

一表通报表常出现“监管填报说明已要求、目标表结构已建、但当前 SQL 尚未加工”的版本差异。agent 不能只凭 SQL 中没有字段就判断“无需报送”，也不能只凭 DDL 中有字段就判断“已经落地”。

必须区分：

- 监管要求字段：外部 `regulatory-knowledge-vault` 填报说明或实体页确认需要报送的字段。
- 本地表结构字段：本仓库 DDL 或表结构材料中已经存在的字段。
- SQL 已加工字段：生成或写入本数据表的 SQL、过程或视图中已经明确赋值、映射或聚合的字段。
- 当前缺口字段：监管要求或表结构已存在，但 SQL 来源、填报规则或是否报送尚未闭环的字段。

报表业务口径页说明“是否需要报送”时，以外部监管来源和实体页为优先依据。数据表页说明“是否已落地”时，同时列出表结构状态和本表自身 SQL 加工状态。血缘页只把 SQL 可追溯字段标为已闭环。

## 数据表页

数据表页是字段口径和表内指标口径的默认承载页。

最低要求：

- 表在监管链路中的角色
- 主键与核心标识
- 关键字段与口径
- 表内指标口径（如有）
- 字段落地状态
- 上下游对象
- 相关来源页和血缘页

如果该表是接口表、结果表或监管报送表，代码类字段必须逐字段写清码值含义、映射规则、特殊值和证据来源，不能只写“见码表”或“参考字典”。

## 血缘页

血缘页默认面向技术人员使用，主要服务于影响分析、反向溯源、字段口径核对和变更评估。

每张血缘页都应包含：

1. 业务链路摘要
2. 节点列表
3. 表级边列表
4. 字段级边列表
5. 条件允许时提供 Mermaid 图
6. 已知缺口、人工假设或未确认点

血缘页不能只停留在“表到表”或“CTE 到 CTE”的粗粒度摘要层。

### 表级血缘

- 说明源表、目标表、中间层、过程或脚本之间的关系。
- 说明每条边对应的主要处理动作，例如过滤、关联、聚合、插入、删除重算。
- 表级图用于快速看全链路，不替代字段级血缘。

### 字段级血缘

对监管报表、标准接口表、核心下游结果表，默认补充字段级血缘。最低要求：

- 覆盖所有目标输出字段。
- 每个目标字段说明来源字段、来源对象、处理逻辑。
- 直接映射写“直接映射”。
- 拼接、条件判断、码值映射、聚合、窗口函数、常量赋值都要显式写出。
- 一个目标字段依赖多个源字段时，要全部列出。

字段级边列表建议包含：源对象、源字段、目标对象、目标字段、处理逻辑、关系类型、证据。

关系类型建议使用：直接映射、条件映射、拼接派生、聚合派生、窗口派生、常量赋值、码值转换。

### Graph 画法

- 建议至少提供总览图和字段级图。
- 字段级图应体现“原表.原字段 -> 目标表.目标字段”。
- 一个 Mermaid 节点只放一个源字段或一个目标字段。
- 一个目标字段依赖多个源字段时，画多条边指向同一个目标字段。
- 字段过多时可以分主题拆成多张 Mermaid 图，但不能省略目标字段。

### 血缘互链

每张血缘页都要显式列出：

- 直接上游对象
- 直接下游对象
- 相关数据表页
- 相关报表业务口径页

每个被血缘页引用的核心数据表页和报表业务口径页，都应回链到对应血缘页。

血缘边必须按 `.agents/rules/血缘双向维护规则.md` 双向落账：

- 源数据表页的“直接下游对象”记录本表被哪些表、报表、SQL 或校验消费。
- 目标数据表页的“直接上游对象”记录本表由哪些来源表、字段或 SQL 加工而来。
- 血缘页记录完整字段级边，数据表页只保留摘要和链接，不重复展开完整字段级血缘。

## 模板联动

创建或更新以下页面时，应同步检查互链是否完整：

- `模板/血缘模板.md`
- `模板/数据表模板.md`
- `模板/报表业务口径模板.md`
- `模板/来源模板.md`
