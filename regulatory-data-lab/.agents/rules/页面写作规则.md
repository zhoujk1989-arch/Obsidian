# 页面写作规则

本文件规定报表业务口径页、数据表页、血缘页和概念页的写作边界。仓库级对象边界以 `AGENTS.md` 为准。

## 通用写作风格

- 业务解释优先使用简洁中文。
- 英文或 `snake_case` 只用于技术标识。
- 优先使用要点和表格。
- 代码片段只保留证明结论所必需的最短部分。
- 不复制大段监管原文。
- 如果页面中出现技术字段名，应同时给出业务名称或中文含义。
- 已存在的仓库内部页面、SQL 文件和外部 vault 来源必须使用 Obsidian wikilink，例如 `[[报表-...]]`、`[[血缘-...]]`、`[[sql/<系统名>/<文件名>.sql|<文件名>.sql]]`。
- 不要把内部对象写成 Markdown 链接、裸路径或仅用反引号包裹的代码路径；反引号只用于字段名、表名、过程名等技术标识。
- 写“未找到”前必须检索对应系统目录；旧来源材料编号、Excel 行号、历史文件名只能作为辅助检索词，不能作为判定页面不存在的唯一依据。
- 巡检时发现“待补充/未找到/待补（目标页名）”但同系统目录下能定位到目标对象的，应直接改为 Obsidian wikilink。
- 不创建指向不存在页面的占位 wikilink；待补对象写普通文本。

## 报表业务口径页

报表业务口径页默认面向业务人员阅读，不要写成 SQL 注释翻译稿或开发说明。

最低要求：

- 说明整张表到底报送哪些业务对象。
- 说明哪些对象纳入、哪些对象排除。
- 说明这是全量报送、增量报送还是截面报送。
- 明细类报表逐字段说明口径。
- 汇总类报表逐指标说明统计对象、过滤条件、聚合方式、分子分母和依赖来源。
- 代码类字段必须写清码值或映射来源。
- 新增报表业务口径页默认按 `模板/报表业务口径模板.md` 落地。

## 数据表页

数据表页是字段口径、码值和表内指标口径的默认承载页；血缘关系、上下游对象、字段落地状态和 SQL 产出状态不写在数据表页，只通过引用入口链接到血缘页。

最低要求：

- 表在监管链路中的角色
- 主键与核心标识
- 关键字段与口径
- 表内指标口径（如有）
- 相关报表业务口径页和血缘页
- 必要证据摘要

不得写入数据表页：

- 字段落地状态、SQL 是否产出、缺 SQL 产出事实等血缘/加工状态
- 直接上游对象、直接下游对象、上下游摘要
- 表级链路、字段级来源、JOIN、过滤、CASE、UNION、聚合等加工链路

上述内容统一写入对应血缘页；数据表页只保留血缘页 wikilink 作为入口。

如果该表是接口表、结果表或监管报送表，代码类字段必须逐字段写清码值含义、映射规则、特殊值和证据来源。

## 血缘页

血缘页默认面向技术人员使用，主要服务于影响分析、反向溯源、字段口径核对和变更评估。

每张血缘页都应包含：

1. 业务链路摘要
2. 节点列表
3. 表级边列表
4. 字段级边列表
5. 条件允许时提供 Mermaid 图
6. 已知缺口、人工假设或未确认点

字段级边列表建议包含：源对象、源字段、目标对象、目标字段、处理逻辑、关系类型、证据。

关系类型建议使用：直接映射、条件映射、拼接派生、聚合派生、窗口派生、常量赋值、码值转换。

## 概念页

概念页是系统级入口和跨对象复用规则的承载页，不是报表、数据表或血缘的第二正文。

允许写入概念页：

- 系统定位、系统边界、系统级对象入口和覆盖摘要。
- 跨报表、跨数据表或跨血缘复用的稳定规则。
- 知识组织方法、索引方法、外部来源引用方法和码值管理方法。
- 系统级 Open Questions，例如整个系统尚未跑数验证、完整清单待补、跨系统映射待确认。

不得写入概念页：

- 单张报表的完整业务口径、字段清单或指标口径。
- 单张数据表的完整字段字典、码值明细或表内指标明细。
- 单条血缘的完整字段级边、SQL 产出状态、上下游对象、JOIN、过滤、CASE、UNION、聚合逻辑。
- 长 SQL、DDL 全文、监管原文或外部 vault 原文摘录。
- 指向不存在对象的占位 wikilink。

新增概念页默认按 `模板/概念模板.md` 落地。系统页命名为 `概念-系统-<系统名>.md`，只做入口、摘要和系统级维护重点；具体对象内容必须回到对应 `报表业务口径/`、`数据表/`、`血缘/` 或 `sql/`。

## 规则联动

本页必须和以下文件一起维护：

- `AGENTS.md`：仓库级知识对象边界、核心原则或目录职责变化时，必须同步检查本页。
- `模板/数据表模板.md`、`模板/报表业务口径模板.md`、`模板/血缘模板.md`、`模板/概念模板.md`：页面章节、引用入口、禁止写入内容或状态判断变化时，必须同步模板。
- `.agents/workflows/数据表-ingest.md`、`.agents/workflows/报表-SQL-ingest.md`、`.agents/workflows/SQL开发.md`、`.agents/workflows/查询与回写.md`、`.agents/workflows/巡检-lint.md`：页面边界变化时，必须同步执行流程。
- `.agents/rules/血缘双向维护规则.md`、`.agents/rules/索引与日志规则.md`：血缘职责、系统入口、互链或日志要求变化时，必须同步规则。
- `日志.md`：本页发生实质规则变更后必须追加记录。

## 模板联动

创建或更新以下页面时，应同步检查互链是否完整：

- `模板/血缘模板.md`
- `模板/数据表模板.md`
- `模板/报表业务口径模板.md`
- `模板/概念模板.md`
