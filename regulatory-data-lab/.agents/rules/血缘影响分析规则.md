# 血缘影响分析规则

目标是基于现有血缘、数据表、报表业务口径和 SQL 文件，给出足够准确、全面、可执行的影响分析。

## 读取顺序

1. `索引.md`
2. `概念/概念-系统-<系统名>.md`
3. 直接相关数据表页
4. 直接相关血缘页
5. 直接相关报表业务口径页
6. `sql/<系统名>/` 下的相关 SQL 文件
7. 必要时检索 `regulatory-knowledge-vault`

如果无法确定系统或对象，先用 `rg` 搜索表名、字段名、中文名、别名、目标字段名和来源字段名。不能只凭一个页面就下结论。

## 影响分类

- 确认影响：有 SQL、字段级血缘、数据表页或报表页明确证明依赖。
- 疑似影响：命名、业务链路或表级关系提示可能依赖，但缺字段级证据。
- 无影响：已有证据证明字段或表不被目标链路消费。
- 待确认：证据不足，无法判定。

## 输出要求

影响分析至少包含：

- 变更对象
- 直接影响对象
- 间接影响对象
- 影响类型
- 严重级别
- 修复对象
- 验证方案
- Open Questions

## 严重级别

| 级别 | 含义 | 示例 |
| --- | --- | --- |
| P0 阻塞 | 会导致生产 SQL/报送链路无法运行，或监管必报字段无法落地 | 删除被 insert/select 直接使用的字段；目标表必填字段无来源 |
| P1 高 | 会导致字段口径、码值、金额或范围明显错误 | 字段含义变更、码值映射变更、过滤条件变化 |
| P2 中 | 会影响文档一致性、血缘完整性或局部解释 | 数据表页、血缘页、报表页未同步 |
| P3 低 | 说明文字、索引入口或非阻塞 Open Questions | 入口缺失、说明过时 |
| Unknown 待确认 | 证据不足，无法判定 | 缺 SQL、缺字段字典、缺外部来源 |

## 修复顺序

1. 先修会导致 SQL 或报送错误的对象。
2. 再更新血缘页和数据表页。
3. 再更新报表业务口径页、系统页和概念页。
4. 最后更新索引和日志。
