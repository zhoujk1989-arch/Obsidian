# 血缘双向维护规则

血缘边必须双向维护：

- 血缘页记录完整边。
- 源数据表页记录“我去到哪”。
- 目标数据表页记录“我从谁来”。

## 可写入依据

只有 SQL、过程、视图、字段字典、用户确认材料、外部 wikilink 或既有知识页能证明的边，才能写为确认血缘；证据不足时写“待确认”或“疑似”，不得硬造。

## 更新顺序

1. 更新或创建血缘页，记录表级边和字段级边。
2. 更新目标数据表页，在“直接上游对象”记录来源对象、来源字段摘要、处理方式和证据。
3. 更新源数据表页，在“直接下游对象”记录消费对象、消费字段摘要、处理方式和证据。
4. 如涉及报表业务口径变化，同步更新报表业务口径页。
5. 如涉及系统级规则或通用概念，同步更新系统页或概念页。
6. 追加 `日志.md`。

## 最低字段级血缘

对监管报表、标准接口表、核心下游结果表，字段级血缘至少覆盖所有目标输出字段。

每个目标字段必须说明：

- 来源对象
- 来源字段
- 处理逻辑
- 关系类型
- 证据状态

无来源字段写“待补血缘”或“无映射来源，待确认”，不得写成已闭环。
