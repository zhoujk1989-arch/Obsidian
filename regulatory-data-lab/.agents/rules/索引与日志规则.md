# 索引与日志规则

## 根索引

`索引.md` 只保留：

- 系统总览
- 项目模块说明
- 任务路由
- 核心概念入口
- 更新检核清单

根索引不平铺具体报表、数据表、血缘或 SQL 文件清单。

## 系统页

每套系统的具体入口维护在 `概念/概念-系统-<系统名>.md`。

系统页应维护：

- 系统定位
- 报表业务口径入口
- 数据表/表族入口
- 血缘入口
- SQL 文件入口
- 当前覆盖摘要
- Open Questions

## 日志

每次 ingest、SQL 开发、查询回写、lint 或规范调整后，应向 `日志.md` 追加记录。

日志至少说明：

- 日期
- 工作类型
- 触达对象
- 关键改动
- 状态判断
- 遗留问题

## 检核清单

1. 系统检核：对象是否归入明确系统；新增系统是否已有系统页。
2. 模块检核：结论是否写入正确模块。
3. 根索引检核：根索引是否只保留系统摘要和模块说明。
4. 系统页检核：具体报表、表族、血缘入口和 SQL 入口是否放在对应系统页。
5. 互链检核：只对已存在页面写 wikilink；待补页面写普通文本。巡检时如发现“待补充/未找到”对象实际已存在，必须直接补成 Obsidian wikilink 并记录。
6. 状态检核：证据不足时保持 `draft`；证据闭环后再评估 `validated`。
7. 日志检核：`日志.md` 追加本次触达对象、状态判断和遗留问题。
