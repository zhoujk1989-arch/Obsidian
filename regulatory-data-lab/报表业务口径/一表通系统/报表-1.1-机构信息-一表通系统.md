- 本页写法：按整表报送范围和字段级口径展开，不以汇总指标为主

## 业务定义

- 这张报表回答的业务问题是什么：截至采集日，一表通需要报送哪些机构，以及这些机构的身份、层级、状态、区划、负责人信息、机构联系电话和科技投入信息分别是什么。
- 统计对象是谁：纳入口径内的银行机构、管理机构、营业机构、虚拟机构和内设机构。
- 统计时点/周期是什么：按 `I_DATE` 生成日报快照，结果落入 `T_1_1`。

## 统计口径

- 纳入口径：
  - 以 `SMTMODS.L_PUBL_ORG_BRA` 当日快照为基础。
  - 允许保留当前失效、但上年末或本年内曾有效的机构。
  - 采集业务范围：上报机构所有分配内部机构号的分支机构及内设机构基本信息，含虚拟机构、理财子公司。
- 排除口径：
  - `ORG_NUM = '999999'`
  - `ORG_NUM` 以 `5`、`6`、`7` 开头
  - 一批指定机构号和瑞祥下属村镇机构号
- 统计粒度：`采集日期 + 机构号`
- 输出维度（本次 ingest 范围 `A010001~A010023`）：
  - 主键与机构标识：`A010001` 机构ID、`A010002` 内部机构号、`A010020` 采集日期
  - 许可证与登记信息：`A010003` 金融许可证号、`A010004` 统一社会信用代码、`A010005` 银行机构名称、`A010006` 支付行号
  - 机构分类：`A010007` 机构类型、`A010008` 机构类别、`A010023` 机构层级
  - 地域和运营：`A010009` 县域机构标识、`A010013` 行政区划、`A010014` 运营状态、`A010015` 成立日期、`A010016` 机构地址、`A010021` 自贸区网点标识
  - 科技金融标识：`A010010` 科技支行标识、`A010011` 科技特色支行标识、`A010012` 科技金融专营机构标识
  - 联系人与统计口径：`A010017` 负责人姓名、`A010018` 负责人工号、`A010019` 负责人联系电话、`A010022` 承办行机构代码
- 版本差异说明：
  - 2.1 版填报说明和本地 DDL 已包含 `A010024~A010026`，但本次 2026-05-13 材料仅覆盖 `A010001~A010023`，`A010024~A010026` 本次不处理。

## 整表报送范围

- 报送对象：
  - 监管集市机构表 `L_PUBL_ORG_BRA` 中满足纳入口径的机构快照记录
  - 包括总行、分行管理机构、营业机构、虚拟机构和内设机构
- 报送目的：
  - 向一表通提供机构基础信息主档，供监管机构识别机构身份、区划、状态和负责人
- 明确不报对象：
  - `ORG_NUM = '999999'`
  - `ORG_NUM` 以 `5`、`6`、`7` 开头的机构
  - 指定剔除机构号清单中的机构
- 报送模式：
  - 日截面全量报送
  - 当天先删后插，以 `A010020=采集日期` 为重算边界

## 字段或指标特殊规则

- 名称修饰规则：
  - `ORG_NUM='990000'` 不改名
  - `ORG_NUM='000000'` 固定显示 `吉林银行股份有限公司(总行)`
  - `ORG_NUM` 末四位 `0000` 补 `(分行管理机构)`
  - 部分末两位 `00` 且满足条件的机构补 `(管理机构)`
- 同名机构规则：
  - 相同 `computed_name` 下，按成立日期最早的一条追加 `(待撤销)`
- 失效保留规则：
  - 当前失效但上年末有效，保留
  - 当前失效但本年内曾有效，保留
- 排除规则：
  - 显式剔除瑞祥下属村镇机构和其他指定机构号
- 版本差异：
  - 2026-05-13 材料覆盖 `A010001~A010023`（23 字段），为本次 ingest 基准
  - 2.1 版填报说明已有 `A010024~A010026`，本地 DDL 已建字段，当前过程未覆盖，本次不处理
  - 后续 agent 判断字段是否"需要报送"时，以外部实体页和填报说明为监管要求依据；判断字段是否"已经落地"时，还必须检查本地 DDL 和 SQL 血缘
