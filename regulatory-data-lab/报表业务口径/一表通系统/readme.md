# 一表通系统报表业务需求目录说明

## 目录职责

本目录保存一表通系统下报表的业务需求加工规则页。

页面用于把一表通相关 SQL 或转换规则翻译成业务人员可评审的自然语言规则，并保留可反向生成 SQL 的加工约束。

## 应包含

- 一表通报表的整表加工目标和报送模式。
- 来源选择、关联条件、过滤条件、去重规则和终态处理。
- 目标字段或指标的加工规则，不写业务含义解释。
- 分户账、机构、客户、交易等场景进入或排除报送范围的加工规则。
- 对应 `sql/一表通系统/` SQL 文件引用。
- `regulatory-knowledge-vault` 中对应报表业务实体 wikilink。

## 不应包含

- 完整监管原文。
- 字段或指标的业务含义解释。
- 数据表完整字段字典。
- 完整字段级血缘。
- 数据表页、血缘页、系统页等本仓库其他知识页的正文引用。
- 只用于 SQL 开发且无法沉淀为加工规则的临时推断。

## 特别约束

涉及分户账类报表时，优先把“是否存在分户账核算/最细一级分户账号”写成可执行的纳入或排除加工规则。
