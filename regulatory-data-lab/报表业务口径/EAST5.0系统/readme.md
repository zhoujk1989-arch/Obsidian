# EAST5.0系统报表业务需求目录说明

## 目录职责

本目录保存 EAST5.0 系统下各张报表的业务需求加工规则页。

页面用于把 EAST5.0 存储过程或 SQL 草案翻译成业务人员可评审的自然语言规则，并保留可反向生成 SQL 的加工约束。

## 应包含

- EAST5.0 报表的整表加工目标和报送模式。
- 来源表选择、JOIN/关联条件、过滤条件、去重规则和终态处理。
- 目标字段或指标的加工规则，不写业务含义解释。
- 码值转换、日期格式转换、金额转换、截取、拼接、默认值和空值处理规则。
- 对应 `sql/EAST5.0系统/` SQL 文件引用。
- `regulatory-knowledge-vault` 中对应报表业务实体 wikilink。

## 不应包含

- EAST5.0 存储过程全文。
- 源表 DDL 全量字段清单。
- 字段或指标的业务含义解释。
- 未核实来源的字段级血缘结论或完整血缘展开。
- 数据表页、血缘页、系统页等本仓库其他知识页的正文引用。
- 非 EAST5.0 主系统对象；跨系统依赖只在正文说明。

## 状态判断

未定位对应 SQL 文件、外部报表业务实体或仍有阻塞性加工规则缺口时，页面保持 `draft`。
