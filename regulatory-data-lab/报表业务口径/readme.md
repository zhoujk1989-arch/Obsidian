# 报表业务口径目录说明

## 目录职责

本目录只保存监管报表的业务口径页，回答“这张报表报什么、报谁、怎么纳入或排除、字段或指标的业务含义是什么”。

## 内容边界

应放入：

- 单张监管报表或稳定报送对象的业务说明。
- 报送范围、纳入排除规则、全量/增量/截面判断。
- 明细字段或汇总指标的业务口径摘要。
- 指向相关数据表页、血缘页、系统页和外部来源的链接。

不应放入：

- 完整字段字典。
- 完整字段级血缘。
- 长 SQL、存储过程全文或跑批脚本。
- 大段监管原文。
- 没有实际页面的占位 wikilink。

## 分层规则

- 必须按系统分目录保存：`报表业务口径/<系统名>/`。
- 新系统先建立或更新 `概念/概念-系统-<系统名>.md`，再新增系统目录。
- 跨系统报表放在主报送系统目录下，并在正文说明涉及的其他系统。

## 命名规则

- 页面文件名使用 `报表-<报表名称或编码>.md`。
- 标题优先使用中文业务名称；技术编码写入 frontmatter 或专门章节。

## 联动要求

新增或大幅更新报表业务口径页时，同步检查对应系统页、相关数据表页、相关血缘页和 `日志.md`。
