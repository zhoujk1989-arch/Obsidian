# SQL目录说明

## 目录职责

本目录保存 agent 生成或维护的 SQL、存储过程、视图、校验脚本和 SQL 开发说明。

## 内容边界

应放入：

- 可追溯的 SQL 草案、修复版、校验 SQL 和加工脚本。
- SQL 文件头部的来源、目标表、字段覆盖、缺口和验证状态摘要。
- 与报表业务口径页、数据表页和血缘页的链接或相对路径说明。

不应放入：

- 无知识页支撑的孤立 SQL。
- 数据表字段字典全文。
- 监管原文全文。
- 临时实验文件。

## 分层规则

- 必须按系统分目录保存：`sql/<系统名>/`。
- 跨系统 SQL 放在主目标系统目录下，并在文件头说明上游系统。

## 命名规则

- 存储过程草案建议使用 `PROC_<系统或报表标识>_<业务缩写>_草案.sql`。
- 校验脚本建议在文件名中包含 `校验` 或 `validation`。

## 联动要求

新增或修复 SQL 后，应同步检查数据表页字段口径、血缘页字段级边、报表业务口径页技术实现摘要、系统页入口和 `日志.md`。
