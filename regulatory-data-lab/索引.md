# regulatory-data-lab 总索引

本页只保留系统层和项目模块层，不列具体报表、数据表、来源页或血缘页。具体对象清单进入对应系统页查看。

## 0. 读取原则

| 场景 | 根索引作用 | 下一层入口 |
| --- | --- | --- |
| 找某套系统的所有内容 | 判断系统边界、当前覆盖和维护重点 | 对应 `概念-系统-<系统名>` |
| 查报表业务口径 | 先定位系统，不在根索引展开报表 | 系统页中的“报表业务口径入口” |
| 查字段含义、码值、落地状态 | 先定位系统和表族 | 系统页中的“表族入口”，再进入数据表页 |
| 查字段来源、SQL 加工链路 | 先定位系统 | 系统页中的“技术追溯入口”，再进入血缘页 |
| SQL 开发或 ingest | 先定位系统和对象类型 | 系统页、来源页、数据表页、报表业务口径页 |
| 巡检、死链、状态核对 | 使用本页模块规则和检核清单 | 系统页、相关对象页、`日志.md` |

索引维护方法见 [[概念-索引体系与检核方法]]。

## 2. 项目模块

| 模块 | 主职责 | 路径 | 根索引如何使用 |
| --- | --- | --- | --- |
| 系统入口 | 承载系统定位、表族/报表族入口、维护重点、系统级 Open Questions | `知识库/概念/概念-系统-<系统名>.md` | 根索引只链接系统页 |
| 报表业务口径 | 说明报表报什么、报谁、纳入排除规则、字段或指标业务口径摘要 | `知识库/报表业务口径/<系统名>/` | 具体报表入口放系统页，不放根索引 |
| 数据表主档 | 承载字段含义、码值、主键、字段落地状态、表内指标和上下游摘要 | `知识库/数据表/<系统名>/` | 根索引只说明模块职责，表族入口放系统页 |
| 来源证据包 | 记录本地 SQL/DDL/字段字典、外部 `regulatory-knowledge-vault` wikilink、检索关键词和关键发现 | `知识库/来源/<系统名>/` | 根索引不列来源页，按系统页或文件路径定位 |
| 血缘 | 说明表级链路、字段级来源、加工逻辑、过滤条件和缺口字段 | `知识库/血缘/<系统名>/` | 只在追技术来源时进入，具体血缘入口放系统页 |
| 概念与方法 | 沉淀跨系统通用规则、系统边界、索引方法和维护约束 | `知识库/概念/` | 根索引保留核心概念入口 |
| 智能体规范 | 约束 agent 的总章程、工作流 SOP、页面写作和索引日志规则 | `AGENTS.md`、`.agents/workflows/`、`.agents/rules/` | 根索引只说明规范分层，执行时按任务读取专题文件 |
| 原始材料 | 保存不可变 SQL、DDL、字段字典和本地技术材料 | `原始材料/SQL/<系统名>/`、`原始材料/表结构/<系统名>/` | 根索引只说明路径，不直接列文件 |
| SQL 开发工作区 | 保存 agent 生成的 SQL 草案、校验 SQL、实现说明 | `工作区/SQL开发/<系统名>/` | 根索引只说明规则，具体草案从系统页或日志追踪 |
| 模板 | 统一知识页和 SQL 草案结构 | `模板/` | ingest 或开发前按对象类型选择模板 |
| 工具 | 外部库检索和 SQL 血缘脚手架 | `工具/` | 需要检索或抽取时调用，不作为知识正文 |
| 日志 | 记录每次 ingest、SQL 开发、lint、索引调整的触达对象和遗留问题 | `日志.md` | 根索引检核完成后同步追加日志 |

## 3. 任务路由

| 任务 | 进入顺序 | 停止条件 |
| --- | --- | --- |
| 回答报表口径 | 根索引定位系统 -> 系统页报表入口 -> 报表业务口径页 | 已能说明报送对象、范围、时间口径、纳入排除规则和字段/指标摘要 |
| 回答字段含义 | 根索引定位系统 -> 系统页表族入口 -> 数据表页 | 已能说明字段含义、取值、码值、落地状态和证据来源 |
| 回答血缘问题 | 根索引定位系统 -> 系统页技术追溯入口 -> 血缘页 | 已能说明源表/源字段、加工逻辑、过滤条件和缺口 |
| 维护血缘上下游 | 根索引定位系统 -> 系统页 -> 血缘页/数据表页 -> 血缘双向维护规则 | 血缘页、源数据表页和目标数据表页均已同步记录来源与去向 |
| 血缘影响分析 | 根索引定位系统 -> 系统页 -> 数据表页/血缘页/SQL/报表页 -> 影响分析规则 | 已能说明确认影响、疑似影响、严重级别、修复对象和验证方案 |
| 开发 SQL | 根索引定位系统 -> 系统页 -> 相关来源页/数据表页/报表页 -> 必要血缘页 | 字段映射可追溯，SQL 草案和校验 SQL 已落到工作区 |
| ingest 新材料 | 根索引定位系统 -> 选择数据表 ingest 或报表 SQL ingest -> 更新对象页 | 来源、数据表/报表、血缘、系统页、日志和状态判断完成 |
| 巡检知识库 | 根索引检核规则 -> 系统页 -> 相关对象页 | 死链、状态、互链、Open Questions 和日志已处理 |

## 4. 扩展规则

- 未来新增 20 多套系统时，根索引每套系统只保留一行摘要。
- 每套系统页负责维护本系统的报表族、表族、来源入口、血缘入口和优先补齐事项。
- 根索引不直接列 2000 多张报表，也不直接列数据表、来源页或血缘页。
- 报表级、字段级、来源级细节全部下沉到系统页和对象页。
- 如果某个系统规模很大，可以在系统页下再拆普通索引页，例如“待补（索引-<系统名>-报表族）”；未创建前不要写成 wikilink。
- 新增系统时，优先创建或更新 `知识库/概念/概念-系统-<系统名>.md`，再回到根索引补系统摘要。

## 6. 更新检核清单

每次 ingest、SQL 开发或 lint 结束前，按以下顺序检查：

1. 系统检核：对象是否归入明确系统；新增系统是否已有系统页。
2. 模块检核：结论是否写入正确模块，报表页不复制血缘，来源页不复制字段字典。
3. 根索引检核：根索引是否只保留系统摘要和模块说明，没有新增具体报表/数据表/来源/血缘清单。
4. 系统页检核：具体报表、表族、来源和血缘入口是否放在对应系统页。
5. 互链检核：只对已存在页面写 wikilink；待补页面写普通文本。
6. 状态检核：只有 DDL 或 SQL 草案未验证时保持 `draft`；证据闭环后再评估 `validated`。
7. 日志检核：`日志.md` 追加本次触达对象、状态判断和遗留问题。
