# EAST5.0系统数据表目录说明

## 目录职责

本目录保存 EAST5.0 系统的源表、接口表、结果表和稳定中间数据集页面，作为表结构事实和字段主档。

## 应包含

- 表级基础信息：系统、表名、中文名、表类型、主题域。
- 表结构事实：字段清单、数据类型、长度、精度、是否非空、主键、索引或唯一约束。
- 字段主档：字段编码、中文名、DDL 注释、格式、是否必填。
- 码值/枚举：代码类字段取值范围、含义、特殊值、空值规则。
- 技术加工状态、SQL 产出状态和上下游对象统一查看对应血缘页。
- 证据摘要：EAST5.0 DDL、字段字典、接口说明、外部实体来源等，不复制全文。
- 引用入口：`sql/EAST5.0系统/` 加工 SQL、对应业务需求页、对应血缘页。

## 不应包含

- 存储过程全文。
- SQL 的完整 JOIN、过滤、CASE、UNION、聚合逻辑。
- 只属于业务需求页的加工规则或业务自然语言需求。
- 上下游摘要信息或完整字段级血缘。
- 未经 DDL、字段字典、接口说明、SQL 产出事实或外部实体支持的字段结论。

## 状态判断

表结构事实、字段主档或码值/枚举存在阻塞缺口时保持 `draft`。SQL 产出状态不在数据表页维护，统一进入对应血缘页。
