# 数据表目录说明

## 目录职责

本目录保存物理表、接口表、结果表或稳定逻辑数据集页面，是表结构事实和字段主档的主承载位置。

页面核心职责：

- 记录表级基础信息、表结构事实和字段清单。
- 记录字段主档、码值/枚举、格式约束、必填约束和字段落地状态。
- 记录证据摘要，但不复制 DDL、字段字典、接口说明或监管原文全文。
- 只引用与本表直接相关的加工 SQL、业务需求页和血缘页，不重复这些页面的内容。

## 内容边界

应放入：

- 表级基础信息：系统、表名、中文名、表类型、主题域。
- 表结构事实：字段清单、类型、长度、精度、是否非空、主键、索引或唯一约束。
- 字段主档：字段编码、中文名、DDL 注释、格式、是否必填。
- 码值/枚举：代码类字段的取值范围、含义、特殊值、空值规则。
- 字段落地状态：本地表结构是否存在、SQL 是否已明确产出、是否存在缺口。
- 证据摘要：DDL、字段字典、接口说明、外部实体来源等，不复制全文。
- 引用入口：加工本表或产出本表的 SQL、对应业务需求页、对应血缘页。

不应放入：

- 整张报表的业务需求加工规则。
- SQL 的完整 JOIN、过滤、CASE、UNION、聚合逻辑。
- 上下游摘要信息或完整字段级血缘。
- 长 SQL、存储过程全文、DDL 全文或接口说明全文。
- 报表级“为什么报、怎么筛选、怎么转 SQL”的自然语言需求。
- 独立字段页、独立指标页或独立码值页。

## 分层规则

- 必须按系统分目录保存：`数据表/<系统名>/`。
- 同名表如果分属不同系统，不得混放。
- 跨系统来源在正文说明，不新建混合目录。

## 命名规则

- 页面文件名使用 `数据表-<表名或业务名>.md`。
- 如果同名对象需要区分，可追加系统名。

## 联动要求

新增或更新数据表页时：

- 页面正文只引用加工 SQL、业务需求页和对应血缘页。
- 不在本页展开 SQL 加工规则、业务需求规则或完整血缘。
- 需要在 `日志.md` 记录本次结构主档或字段主档变更、状态判断和遗留问题。
