# 数据表目录说明

## 目录职责

本目录保存物理表、接口表、结果表或稳定逻辑数据集页面，是字段口径和表内指标口径的主承载位置。

## 内容边界

应放入：

- 表在监管链路中的角色。
- 主键、核心标识和关键字段口径。
- 码值含义、取值范围、映射规则和特殊值。
- 字段落地状态、上下游对象和必要证据摘要。
- 指向报表业务口径页、血缘页、SQL 文件和系统页的链接。

不应放入：

- 整张报表业务背景。
- 复杂字段级血缘全量展开。
- 长 SQL 或存储过程全文。
- 独立字段页、独立指标页或独立码值页。

## 分层规则

- 必须按系统分目录保存：`数据表/<系统名>/`。
- 同名表如果分属不同系统，不得混放。
- 跨系统来源在正文说明，不新建混合目录。

## 命名规则

- 页面文件名使用 `数据表-<表名或业务名>.md`。
- 如果同名对象需要区分，可追加系统名。

## 联动要求

新增或更新数据表页时，同步检查相关血缘页的来源/去向、报表页的字段或指标引用、系统页入口和 `日志.md`。
