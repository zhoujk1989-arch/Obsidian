# EAST5.0系统血缘目录说明

## 目录职责

本目录保存 EAST5.0 系统数据表的表级上下游、字段级血缘和代码字段加工使用摘要。

## 应包含

- 当前 EAST5.0 表标识：系统、表名、中文名、表类型。
- SQL 证据：加工本表或消费本表的 `sql/EAST5.0系统/` 文件、过程、视图。
- 表级上游关系：上游表到当前表。
- 表级下游关系：当前表到下游表。
- 字段级血缘：源表、源字段、目标表、目标字段、处理逻辑、关系类型、SQL 定位、确认状态。
- 代码字段加工使用摘要：码值转换、CASE 分支、默认值、特殊值、异常处理和 SQL 定位。
- 血缘缺口：缺 SQL、缺源字段、缺目标字段、缺码值规则、缺确认。

## 不应包含

- 存储过程全文。
- 结果表字段字典全文。
- EAST5.0 报表业务信息或报送口径解释。
- 业务需求页里的加工规则全文。
- 缺乏 SQL 证据的字段级边。

## 状态判断

只完成草案抽取但未完成表级上下游、字段级血缘和代码字段加工摘要核对时保持 `draft`。确认 SQL 证据、字段级边和缺口状态后，才可评估 `validated`。
