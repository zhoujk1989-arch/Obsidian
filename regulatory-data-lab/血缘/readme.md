# 血缘目录说明

## 目录职责

本目录保存数据表之间的血缘关系页面，是表级上下游、字段级来源去向和代码字段加工使用摘要的主承载位置。

页面核心职责：

- 统计每一张表的直接上游表和直接下游表。
- 记录目标字段来自哪些源表、源字段，以及字段之间的处理关系。
- 记录代码字段在加工过程中的转换、默认值、特殊值和异常处理摘要。
- 记录血缘缺口和待确认点。
- 只存储血缘信息，不承载报表业务信息。

## 内容边界

应放入：

- 当前表标识：系统、表名、中文名、表类型。
- SQL 证据：加工本表或消费本表的 SQL 文件、过程、视图。
- 表级上游关系：上游表到当前表的依赖边。
- 表级下游关系：当前表到下游表的消费边。
- 字段级血缘：源表、源字段、目标表、目标字段、处理逻辑、关系类型、SQL 定位、确认状态。
- 代码字段加工使用摘要：代码字段、来源代码、转换逻辑、输出值、默认值、异常处理、SQL 定位。
- 血缘缺口：缺 SQL、缺源字段、缺目标字段、缺码值规则、缺确认。

不应放入：

- 报表业务信息。
- 报表为什么报、报谁、怎么报。
- 字段业务含义。
- 数据表完整字段字典。
- 完整 SQL、存储过程或视图全文。
- 监管原文。
- 业务需求页里的加工规则全文。
- 数据表页里的结构事实全文。
- 无 SQL 或可确认加工链路支持的伪闭环字段结论。

## 分层规则

- 必须按系统分目录保存：`血缘/<系统名>/`。
- 跨系统链路放在主目标系统目录下，并显式说明涉及系统。

## 命名规则

- 页面文件名使用 `血缘-<目标表或链路对象>.md`。

## 联动要求

维护血缘边时：

- 血缘页是表级上下游和字段级血缘的主记录位置。
- 数据表页只保留对应血缘页引用，不回写上下游摘要。
- 业务需求页不承载血缘事实。
- 需要在 `日志.md` 记录本次血缘关系、代码字段加工摘要和遗留缺口。
