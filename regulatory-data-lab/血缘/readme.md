# 血缘目录说明

## 目录职责

本目录保存端到端血缘页面，服务影响分析、反向溯源、字段口径核对和变更评估。

## 内容边界

应放入：

- 业务链路摘要。
- 节点列表、表级边和字段级边。
- 加工逻辑、关系类型、证据来源和缺口字段。
- 必要时提供 Mermaid 图。
- 源数据表页和目标数据表页的双向维护链接。

不应放入：

- 监管业务定义的主解释。
- 数据表完整字段字典。
- 长 SQL 全文。
- 无 SQL 或知识页支持的伪闭环字段结论。

## 分层规则

- 必须按系统分目录保存：`血缘/<系统名>/`。
- 跨系统链路放在主目标系统目录下，并显式说明涉及系统。

## 命名规则

- 页面文件名使用 `血缘-<链路或目标对象>.md`。

## 联动要求

维护血缘边时必须双向同步：血缘页记录完整边，源数据表页记录“我去到哪”，目标数据表页记录“我从谁来”。
