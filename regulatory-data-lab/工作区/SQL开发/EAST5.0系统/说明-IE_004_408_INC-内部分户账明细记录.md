## 5. 2026-05-05 重构要点

- 依据原始业务需求《023_内部分户账明细记录.md》逐项校准 32 个字段映射。
- 消除 3 个 JOIN TODO：
  - `T_7_10 src LEFT JOIN IE_004_407 acct ON src.G100002 = acct.NBFHZZH`（内部分户账 enrich）
  - `LEFT JOIN IE_004_402 dim_mx ON src.G100007 = dim_mx.KJKMBH`（明细科目名称）
  - `LEFT JOIN IE_004_402 dim_dfk ON src.G100023 = dim_dfk.KJKMBH`（对方科目名称）
- 实现 WHERE 过滤：`T_7_10.G100028 = V_DATA_DATE`（采集日期过滤）
- 补齐 5 个码值 CASE 转换：
  - JYLX（交易类型）：15 个精确分支 + 1 个 `00%` 通配 + ELSE
  - JYQD（交易渠道）：8 个精确分支 + 1 个 `07%` 通配 + 1 个 `00%` 通配 + ELSE
  - JYJDBZ（交易借贷标志）：3 个精确分支 + ELSE ''
  - CBMBZ（冲补抹标志）：2 个精确分支 + ELSE
  - XZBZ（现转标志）：2 个精确分支 + ELSE ''
- 日期格式转换：HXJYRQ/JZRQ/XZRQ 使用 `DATE_FORMAT(col, '%Y%m%d')`；HXJYSJ 使用 `REPLACE(DATE_FORMAT(col, '%H:%i:%s'), ':', '')`
- 柜员号处理：JYGYH/SQGYH 对 `'自动'` 转 NULL
- 3 个缺口字段 GSFZJG/SENSITIVEFLAG/DFKHLB 在 SQL 中置 NULL，符合审计处置原则
- 校验 SQL 从 4 项扩展至 21 项（码值检查、日期格式检查、源表覆盖检查等）
