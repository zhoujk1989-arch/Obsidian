## 4. 码值转换规则

### 4.1 机构类别（JGLB）

| 源值（A010008） | 目标值 |
| --- | --- |
| 0101、0102 | 管理机构 |
| 0201、0202、0203 | 营业机构 |
| 0301、0302 | 虚拟机构 |
| 0401、0402 | 内设机构 |
| 其他 / NULL | NULL（异常） |

### 4.2 营业状态（YYZT）

| 源值（A010014） | 目标值 |
| --- | --- |
| 01 | 营业 |

| 其他 | 停业（默认） |

## 5. 表级关联逻辑

```
T_1_1（当日机构快照）
  LEFT JOIN T_1_1（上月末机构快照）→ 连续停业判断
  LEFT JOIN T_1_3（员工表）→ 补负责人职务
  LEFT JOIN (T_4_1 UNION T_4_3) → 回补有余额/有分户账的机构
  WHERE 过滤：排除连续停业且无回补的机构
```

### 5.1 去重策略

- **机构信息（T_1_1）**：同一内部机构号取最大采集日期下的最大版本号（A010001 最大）。
- **员工信息（T_1_3）**：同一员工取最新采集日期（A030028 最大），次级取最大版本号（A030002 最大）。
- **上月末快照**：与当日快照相同去重逻辑。

### 5.2 过滤条件

```sql
WHERE NOT (
    prev.A010014 IN ('00', '02', '03')  -- 上月停业或被合并
    AND org.A010014 IN ('00', '02', '03')  -- 本月仍停业或被合并
    AND supplement.org_id IS NULL  -- 且无总账余额或内部分户账记录
)
```

## 6. 关键开发说明

### 6.1 GBase 8a MPP 日期转换

由于 GBase 8a MPP 不支持 MySQL 的 `DATE_FORMAT()` 和 `STR_TO_DATE()`，使用以下方案：

- **YYYYMMDD → DATE**：`CAST(CONCAT(LEFT(str,4),'-',SUBSTR(str,5,2),'-',SUBSTR(str,7,2)) AS DATE)`
- **DATE → YYYYMMDD**：`CONCAT(CAST(YEAR(dt) AS VARCHAR(4)), LPAD(CAST(MONTH(dt) AS VARCHAR(2)),2,'0'), LPAD(CAST(DAY(dt) AS VARCHAR(2)),2,'0'))`

> 注意：`YEAR()`、`MONTH()`、`DAY()`、`DATE_SUB()`、`LAST_DAY()` 函数的具体兼容性需按现场 GBase 8a MPP 版本确认。

### 6.2 异常处理

- 过程使用 `EXIT HANDLER FOR SQLEXCEPTION` 捕获异常。
- 异常时记录错误日志（占位）、回滚事务、重新抛出异常。
- 参数校验失败时直接 `SIGNAL SQLSTATE` 终止执行。

### 6.3 重跑安全

- 清理区仅删除 `JGXXB` 中 `CJRQ = P_DATA_DATE` 的记录，不影响其他批次数据。
- 清理与插入在同一事务中，保证原子性。
