# 概念-系统-监管集市系统

## 系统定位

- 监管集市系统是 regulatory-data-lab 中独立维护的一个系统边界。
- 该系统下的 SQL、数据表和来源页应放在 `监管集市系统` 对应目录下维护。
- 当前主要作为一表通和 EAST5.0 下游加工的机构主数据及码值证据入口。

## 当前覆盖范围

| 对象 | 数量 | 状态摘要 |
| --- | ---: | --- |
| 数据表页 | 1 | 已建立机构主数据表字段主档 |
| 来源页 | 2 | 已建立机构表来源页和码值字典证据入口 |
| 报表业务口径页 | 0 | 当前不作为报表系统维护 |
| 血缘页 | 0 | 下游使用关系主要在一表通/EAST5.0 血缘中表达 |

## 表族入口

| 表族 | 代表页 | 维护重点 |
| --- | --- | --- |
| 机构主数据 | [[数据表-机构表-l_publ_org_bra-监管集市系统]] | 机构地址、机构许可、机构层级和下游映射 |
| 码值证据入口 | [[来源-监管集市系统-码值字典]] | 仅作为证据入口，新增码值说明优先写入消费方数据表字段口径 |

## 维护规则

- 先维护系统内表页和来源页。
- 如果监管集市字段是代码类字段，在被下游接口表使用时，优先把码值说明写进下游接口表字段口径。
- 只有当系统边界、字段角色或通用术语需要稳定说明时，才补充概念页。
- 下游一表通或 EAST5.0 加工关系应写在下游系统血缘页，不在监管集市目录制造跨系统混合页。
