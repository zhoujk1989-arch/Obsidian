## 业务定义

- 本表报送银行机构所有对公客户的基本信息。
- 统计对象：对公客户，包括单一法人客户、同业客户、集团客户。
- 统计时点：截至采集日有效的数据，以及上一采集日至采集日期间结清、失效、终结等所有视为终态的数据。

## 统计口径

### 纳入规则

- 单一法人客户：银行对企业分公司等非法人机构授信的，视同单一法人客户报送。
- 同业客户：范围与银保监会非现场监管统计一致。
- 集团客户：纳入集团统一授信的客户，虚拟集团客户以及成员单位均填报为集团客户。
- 个体工商户、私营业主以机构名义登记客户资料的应纳入本表填报范围。
- 集团类的虚拟客户的客户信息要上报。

### 排除规则

- 不再列为银行客户的数据不报送。

### 允许为空

- 同业客户、政府机关、境外客户，若无法获取相关信息无法填报的数据项，允许为空。

### 统计粒度

- 以客户统一编号 + 采集日期为一条记录。

### 输出维度

- 客户类型：单一法人客户 / 同业客户 / 集团客户

## 整表报送范围

- 本表覆盖三大类对公客户：单一法人客户、同业客户、集团客户。
- 报送模式为全量表，报送截至采集日有效的数据，以及上一采集日至采集日期间结清、失效、终结等所有视为终态的数据。
- 不报送不再列为银行客户的数据。

## 技术实现摘要

- 主要源表：一表通单一法人基本情况（T_2_1）、同业客户基本情况（T_2_3）、集团基本情况（T_2_2）、集团成员名单（T_3_3）、机构信息（T_1_1）。
- 主要过滤条件：采集日期 = 跑批日期；集团客户通过母公司客户ID关联单一法人取证件信息。
- 主要加工逻辑：三大客户类型 UNION ALL 合并；证件类别按优先级取数；企业分类按对公客户类型码值转换；日期格式 YYYY-MM-DD → YYYYMMDD。
