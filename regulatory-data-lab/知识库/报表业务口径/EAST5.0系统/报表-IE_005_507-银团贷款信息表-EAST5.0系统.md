# 数据表-T_6_5-银团贷款协议-一表通系统

## 页面边界

- 本页面向业务人员，说明 `银团贷款协议`（T_6_5）的业务含义、字段口径和下游消费情况。
- 完整字段类型和维护信息维护在 [[数据表-T_6_5-银团贷款协议-一表通系统]]。
- 本页依据业务需求文档和 SQL 草案整理，尚未经过跑数验证，状态保持 `draft`。

## 系统归属

- 所属系统：一表通系统
- 是否跨系统：是，目标为 EAST5.0。

## 业务定义

- 本表报送对象：银（社）团贷款信息。以信贷借据号为最小颗粒报送，同一合同下多笔借据的按多条报送。
- 本表回答的问题：在采集期内，按 EAST5.0 要求提交 `银团贷款信息表` 对应业务对象的明细信息。
- 统计时点/周期：以 `采集日期`（P_DATA_DATE）为准；按月采集，采集日期为月末最后一天。

## 统计口径

- 纳入口径：银（社）团贷款信息。以信贷借据号为最小颗粒报送，同一合同下多笔借据的按多条报送。
- 排除口径：已结清、核销的数据在次月不再报送；当月内结清/核销的需在采集日当期报送。
- 输出维度：目标 EAST5.0 表 `IE_005_507` 的 27 个字段。
- 依赖关系：本表由一表通来源表 `T_6_5`（银团贷款协议）和 EAST 借据表 `IE_005_504`（对公信贷业务借据表）按字段映射、关联、过滤、码值转换和日期格式转换生成。

## 整表报送范围

- 报送范围来源于 `原始材料/业务需求/EAST5.0/034_银团贷款信息表.md`。
- 报送模式为：全量表，截至采集日有效的数据 + 上一采集日至采集日期间结清/失效/终结等终态数据。
- 表级规则摘要：取日期在当月且通过信贷合同号关联生成 EAST 对公信贷业务借据表来筛选范围。

## 技术实现摘要

- SQL 草案：`工作区/SQL开发/EAST5.0系统/PROC_EAST_IE_005_507_YTDKXXB_草案.sql`
- 主要处理：按采集日期重跑（先删后插），LEFT JOIN T_6_5 和 IE_005_504，完成直接映射、日期格式转换、码值转换（YTCYLX）、金额按币种逻辑处理和备注处理。
- 关联键：T_6_5.F050001 = IE_005_504.XDHTH（协议ID = 信贷合同号）。
- 币种计算：子查询统计同一合同下借据币种一致性，决定 BZ 字段值。
- 终态纳入：通过 IE_005_504.DKZT（非结清/核销）和 ZJRQ（终结日期 >= 上一采集日）判断。
