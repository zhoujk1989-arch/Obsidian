# 报表-IE_005_508-票据贴现表-EAST5.0系统

## 页面边界

- 本页维护 `票据贴现表` 的业务口径说明，包括报送范围、纳入排除规则、字段映射和加工逻辑。
- 数据表定义见 [[数据表-IE_005_508-票据贴现表-EAST5.0系统]]；血缘见 [[血缘-IE_005_508-票据贴现表-EAST5.0系统]]。

## 业务背景

- `IE_005_508` 是 EAST5.0 `票据贴现表`，用于采集填报机构对客户办理的商业票据贴现业务信息。
- 不包括对金融机构办理的买断式转贴现业务。

## 报送范围与纳入排除规则

- **纳入**：填报机构对客户办理的商业票据贴现。
- **排除**：对金融机构办理的买断式转贴现业务。
- **终态规则**：票据状态为"卖断"、"解付"或"核销"的数据，在报送票据最后状态的次月可不再报送。
- **表级规则**：取日期在当月且剔除上月状态为失效的数据。

## 技术实现摘要

- **SQL 草案**：`工作区/SQL开发/EAST5.0系统/PROC_EAST_IE_005_508_PJTXB_草案.sql`
- **源表**：T_6_13（票据协议）为主来源，T_1_1（机构信息）为关联来源
- **JOIN 条件**：T_1_1.A010001 = T_6_13.F130003（机构ID）
- **WHERE 过滤**：T_6_13.F130049 = P_DATA_DATE（当月采集日期）
- **码值转换**：PJLX（票据类型）、PJZT（票据状态）使用 CASE 映射
- **日期转换**：CJRQ/PJDQRQ/PJCPRQ/TXRQ 使用 DATE→YYYYMMDD 格式转换
- **加工映射**：NBJGH 使用 SUBSTR(F130003, 12)
- **SQL 草案状态**：draft，尚未在 GBase 环境执行验证
