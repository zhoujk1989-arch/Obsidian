## 技术实现摘要

- SQL 草案：`工作区/SQL开发/EAST5.0系统/PROC_EAST_IE_007_705_INC_DKWJXTBDB_草案.sql`
- 主要处理：按采集日期重跑，关联一表通来源表 T_8_12（五级分类状态）作为主表，LEFT JOIN T_1_1（机构信息）、IE_004_411（对公信贷分户账）、IE_004_409（个人信贷分户账）、IE_008_801（信用卡信息表）、IE_002_203（对公客户信息表）、IE_002_201（个人基础信息表）完成全部18个字段映射。
- 表级规则：采集日期在跑批日期当月内 且 采集日期=调整日期；排除新发放业务（原五级分类为空或'00'的记录）。
- 码值转换：五级分类码值透传（01/02/03/04/05/00）、变动方式码值透传（01人工/02自动）、经办人工号自动变动置空。
- 信用卡 XDJJH 特殊处理：COALESCE(NULLIF(H120002,''), cc.KH)。
- 缺口字段处理：GSFZJG/KHLB/SENSITIVEFLAG 从 EAST 关联表获取。
- 2026-05-09 重构校准要点：移除 WHERE 1=1 占位、T_1_1 LEFT JOIN 补齐 A010020 采集日期过滤防止重复行。
