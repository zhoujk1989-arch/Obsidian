## 业务定义

- 本表报送对象：本表报送信贷资产转让合同与被转让贷款借据的对应关系，一个转让合同对应多个借据的，按借据填报多条记录。对于信用卡资产转让，以信用卡账号填报为借据号。票据的买卖、买入返售、卖出回购不在本表填报。
- 本表回答的问题：在采集期内，按 EAST5.0 要求提交 `资产转让关系表` 对应业务对象的明细信息。
- 统计时点/周期：以 `采集日期` 或业务需求中的报告期规则为准；全量表按采集日期重跑。

## 统计口径

- 纳入口径：本表报送信贷资产转让合同与被转让贷款借据的对应关系，一个转让合同对应多个借据的，按借据填报多条记录。对于信用卡资产转让，以信用卡账号填报为借据号。票据的买卖、买入返售、卖出回购不在本表填报。
- 排除口径：票据的买卖、买入返售、卖出回购不在本表填报。
- 输出维度：目标 EAST5.0 表 `IE_007_704` 的字段清单。
- 依赖关系：本表由一表通源表按字段映射、关联、过滤、码值转换和日期格式转换生成。

## 整表报送范围

- 报送范围来源于 `原始材料/业务需求/EAST5.0/047_资产转让关系表.md`。
- 报送模式为：全量表，报送截至采集日有效的数据，以及上一采集日至采集日期间结清、失效、终结等所有视为终态的数据。
- 表级规则（2026-05-09 重构后已实现）：
  - 主表：【一表通】【信贷资产转让】(T_7_9)，取按照【借据ID】、【资产转让方向】分组，采集日期降序排列后的第一条数据
  - 过滤条件：【信贷资产转让】.采集日期<=跑批日期
  - 左关联：【一表通】【机构信息】(T_1_1)：SUBSTR(TRIM(T_7_9.机构ID),12)=SUBSTR(TRIM(T_1_1.机构ID),12) AND T_1_1.采集日期=跑批日期
  - 内关联：【一表通转EAST】【信贷资产转让表】(IE_007_703)：IE_007_703.采集日期=跑批日期 AND IE_007_703.转让合同号=T_7_9.协议ID AND IE_007_703.资产转让方向=CASE T_7_9.资产转让方向 WHEN '01' THEN '转入' WHEN '02' THEN '转出' ELSE '' END

## 技术实现摘要

- SQL 草案：`工作区/SQL开发/EAST5.0系统/PROC_EAST_IE_007_704_ZCZRGXB_草案.sql`
- 校验 SQL：`工作区/SQL开发/EAST5.0系统/CHECK_EAST_IE_007_704_ZCZRGXB_校验.sql`
- 主要处理（2026-05-09 重构后）：
  - 窗口去重：按借据ID (G090003) + 资产转让方向 (G090005) 分组，采集日期 (G090018) 降序取 rn=1
  - WHERE 过滤：T_7_9.采集日期 <= 跑批日期
  - LEFT JOIN T_1_1：按机构ID从第12位截取关联，取金融许可证号 (JRXKZH)
  - INNER JOIN IE_007_703：按转让合同号+资产转让方向关联，筛选已转换记录
  - 码值 CASE：XDZCLX '01'→'个人贷款'/'02'→'对公贷款'/'03'→'信用卡贷款'/ELSE→'其他'
  - 日期转换：CJRQ 直接赋参数 P_DATA_DATE (YYYYMMDD)
  - 金额转换：ZRDKLX/ZRDKBJ 使用 CAST(NULLIF(TRIM(...),'') AS DECIMAL(20,2))
  - 2个缺口字段（SENSITIVEFLAG/GSFZJG）置 NULL
