# 概念-监管报表知识主目录

## 定义

- 这是一个由 LLM 持续维护的监管报表知识主目录。
- 本库当前只保存结构化语义层：报表业务口径、数据表字段主档、血缘、SQL、概念和模板。
- 监管原文、报表说明、主题规则和报表清单不保存在本库，统一关联外部知识库 `regulatory-knowledge-vault`。
- 目标是让人先读 wiki，再回看 SQL，而不是每次都从 SQL 原文重新理解。

## 为什么需要它

- 监管报表经常跨多个源表和中间层，口径理解成本高。
- 单纯给 SQL 加注释，难以沉淀跨报表复用知识。
- 业务、数据、监管三方需要共享同一套可追溯定义。
- 原始监管文字精度要求高，更适合在独立资料库中维护，本库只沉淀结构化语义层。

## 成功标准

- 一个业务人员能通过 `报表业务口径/` 看懂报表在算什么。
- 一个数据人员能通过 `血缘/` 追到指标来自哪里。
- 一个监管报表维护人员能知道口径变动会影响哪些报表和指标。
- 一个分析人员能通过页面中的外部来源 wikilink 追到 `regulatory-knowledge-vault` 中对应的原文和规则页。
