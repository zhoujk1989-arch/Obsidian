# 概念目录说明

## 目录职责

本目录保存跨报表复用的概念、系统页、方法页和仓库知识组织规则。

## 内容边界

应放入：

- 系统页：`概念-系统-<系统名>.md`。
- 通用术语、业务边界、码值管理方法、索引体系和检核方法。
- 可跨报表复用的稳定结论。
- 各系统的对象入口、覆盖摘要和 Open Questions。

不应放入：

- 单张报表的明细字段清单。
- 单张数据表的完整字段字典。
- 单条血缘的完整字段级边。
- SQL 文件或长监管原文。

## 命名规则

- 概念页文件名使用 `概念-<主题>.md`。
- 系统页文件名固定为 `概念-系统-<系统名>.md`。

## 联动要求

新增系统或新增具体对象时，优先维护对应系统页；根 `索引.md` 只保留系统摘要，不平铺明细入口。
