# 概念目录说明

## 目录职责

本目录保存跨报表复用的概念、系统页、方法页和仓库知识组织规则。

概念页是“地图和规则”，不是“对象正文”。具体业务口径、字段主档、字段级血缘和 SQL 实现必须回到对应对象目录。

## 内容边界

应放入：

- 系统页：`概念-系统-<系统名>.md`。
- 通用术语、业务边界、码值管理方法、索引体系和检核方法。
- 可跨报表复用的稳定结论。
- 各系统的对象入口、覆盖摘要、维护重点和系统级 Open Questions。

不应放入：

- 单张报表的明细字段清单。
- 单张数据表的完整字段字典。
- 单条血缘的完整字段级边。
- 字段落地状态、SQL 产出状态、上下游对象或加工链路。
- SQL 文件、长 SQL、DDL 全文或长监管原文。

## 命名规则

- 概念页文件名使用 `概念-<主题>.md`。
- 系统页文件名固定为 `概念-系统-<系统名>.md`。
- 新增概念页默认使用 `模板/概念模板.md`。

## 联动要求

新增系统或新增具体对象时，优先维护对应系统页；根 `索引.md` 只保留系统摘要，不平铺明细入口。

如果新增内容只影响单张报表、单张数据表或单条血缘，不回写概念页；只有影响系统入口、复用规则、系统边界、索引方法或跨对象稳定结论时，才维护概念页。
