# 索引与日志规则

`索引.md` 是总索引，不是全量对象目录。它只保留系统层和项目模块层。

## 根索引职责

`索引.md` 只承载：

- 系统总览
- 项目模块说明
- 任务路由
- 扩展规则
- 核心概念入口
- 更新检核清单

根索引不承载：

- 具体报表清单
- 具体数据表清单
- 具体来源页清单
- 具体血缘页清单
- 大批量 SQL 文件清单

未来新增 20 多套系统、2000 多张报表时，根索引每套系统只保留一行摘要。

## 系统页职责

每套系统的具体入口维护在 `知识库/概念/概念-系统-<系统名>.md`。

系统页负责：

- 系统定位
- 当前覆盖范围
- 报表业务口径入口
- 表族或报表族入口
- 来源定位方式
- 技术追溯入口
- 维护规则
- 系统级 Open Questions

如果某系统规模很大，可以在系统页下再拆普通索引页，例如“待补（索引-<系统名>-报表族）”。未创建前不要写成 wikilink。

## 更新规则

新增系统时：

1. 创建或更新 `知识库/概念/概念-系统-<系统名>.md`。
2. 在 `索引.md` 的系统总览中新增一行系统摘要。
3. 在 `日志.md` 记录系统初始化。

新增具体报表时：

1. 创建或更新报表业务口径页。
2. 更新对应系统页的报表入口或报表族摘要。
3. 不把具体报表写入根索引。
4. 更新 `日志.md`。

新增具体数据表时：

1. 创建或更新数据表页。
2. 更新对应系统页的表族入口或覆盖范围。
3. 不把具体数据表写入根索引。
4. 更新 `日志.md`。

新增来源页或血缘页时：

1. 更新对应对象页之间的互链。
2. 更新对应系统页的来源定位或技术追溯入口。
3. 不把具体来源或血缘写入根索引。
4. 更新 `日志.md`。

## 日志规则

每次 ingest、SQL 开发、查询回写、lint、血缘影响分析、模板调整、规范调整或索引调整后，应向 `日志.md` 追加时间记录。

日志至少说明：

- 工作类型：`ingest`、`sql-dev`、`query`、`lint`、`impact-analysis`、`template`、`refactor`、`index` 等。
- 触达对象：新增或更新了哪些来源页、报表业务口径页、数据表页、血缘页、概念页、SQL 草案或系统页。
- 证据来源：本地 SQL/DDL、外部 wikilink、用户澄清或既有知识页。
- 状态判断：哪些页面保持 `draft`，哪些页面可评估为 `validated`。
- 遗留问题：未确认字段、码值、外部来源、血缘缺口、SQL 验证缺口。

日志可以引用具体页面作为历史记录；日志中的历史占位 wikilink不纳入知识图完整性判断。但新增知识页正文中不得制造不存在 wikilink。

## 检核清单

每次工作结束前检查：

1. 系统检核：对象是否归入明确系统；新增系统是否已有系统页。
2. 模块检核：结论是否写入正确模块，报表页不复制血缘，来源页不复制字段字典。
3. 根索引检核：根索引是否只保留系统摘要和模块说明。
4. 系统页检核：具体报表、表族、来源和血缘入口是否放在对应系统页。
5. 互链检核：只对已存在页面写 wikilink；待补页面写普通文本。
6. 状态检核：只有 DDL 或 SQL 草案未验证时保持 `draft`；证据闭环后再评估 `validated`。
7. 日志检核：`日志.md` 是否记录触达对象、状态判断和遗留问题。
