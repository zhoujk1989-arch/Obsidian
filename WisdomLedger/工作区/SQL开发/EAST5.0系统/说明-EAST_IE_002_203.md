# 说明-EAST IE_002_203 对公客户信息表存储过程

## 目标对象

- 存储过程：`PROC_EAST_IE_002_203_DGKHXXB`
- 目标表：`IE_002_203`（对公客户信息表，数据表编号 203）
- 数据粒度：以客户统一编号 + 采集日期为一条记录

## 源表及角色

| 源表 | 一表通表名 | 角色 |
| --- | --- | --- |
| T_2_1 | 单一法人基本情况 | 单一法人客户主源；集团客户母公司信息源 |
| T_2_3 | 同业客户基本情况 | 同业客户主源 |
| T_2_2 | 集团基本情况 | 集团客户主源 |
| T_3_3 | 集团成员名单 | 集团客户成员关联（首次信贷关系、员工人数汇总） |
| T_1_1 | 机构信息 | 补金融许可证号、银行机构名称 |

## 主连接键与时间条件

- 时间条件：所有源表以采集日期 = 跑批日期过滤
- 单一法人：`T_2_1.B010060 = P_DATE`
- 同业客户：`T_2_3.B030036 = P_DATE`
- 集团客户：`T_2_2.B020019 = P_DATE`
- 集团成员：`T_3_3.C030010 = P_DATE`
- 母公司关联：`T_2_1.B010001 = T_2_2.B020020` 且 `T_2_1.B010060 = P_DATE`
- 机构信息：`T_1_1.A010020 = P_DATE`，取最新快照（NOT EXISTS 去重）

## 关键过滤条件

- 三大客户类型通过 UNION ALL 合并，不额外过滤
- 机构信息取最新快照：NOT EXISTS 取同机构ID最小 A010001 的记录
- 集团客户成员首次信贷关系取 MIN 最早日期
- 集团客户员工人数取 SUM 汇总

## 日期格式转换

- 源表日期格式：`YYYY-MM-DD`
- 目标日期格式：`YYYYMMDD`
- 转换方式：`TO_CHAR(TO_DATE(col, 'YYYY-MM-DD'), 'YYYYMMDD')`
- 首次建立信贷关系年月：源为 `YYYY-MM`，截取前 6 位得 `YYYYMM`

## 特殊逻辑说明

### 集团客户首次建立信贷关系年月

通过 T_3_3 集团成员名单关联 T_2_1 单一法人和 T_2_3 同业客户，
取所有成员中最早的 YYYYMM 日期。

### 集团客户员工人数

通过 T_3_3 集团成员名单关联 T_2_1 单一法人，
汇总所有成员的员工人数 B010041。

### 内部机构号

从机构ID（B010002/B030002/B020002）第 12 位开始截取。

## 未确认点

1. **纳入范围过滤**：需求文档要求以 20+ 个 EAST 表的客户统一编号做 UNION ALL
   去重后作为纳入范围，但这些 EAST 表多数尚未落地。当前实现不对一表通源做
   客户编号过滤。若后续需要，可在 UNION ALL 之后加 `WHERE KHTYBH IN (子查询1)`。

2. **采集日期格式**：若源表采集日期字段已是字符串格式（如 '2026-04-28'），
   `TO_DATE(col, 'YYYY-MM-DD')` 应能正确解析；若源表为其他格式需调整。

3. **证件类别码值完整性**：一表通"其他证件类型"字段（B010062/B030039）的
   码值范围可能与需求文档中的映射不完全一致，需跑数后验证。

4. **涉密客户处理**：需求文档提到境内涉密机构客户名称填报为"*********"，
   当前未实现涉密标志 SENSITIVEFLAG 和名称变形逻辑，GSFZJG 和 SENSITIVEFLAG
   字段当前置空。

5. **集团客户成员去重**：若同一成员ID在 T_3_3 中有多条记录（不同采集日期），
   当前通过采集日期过滤已保证只取当期数据。
