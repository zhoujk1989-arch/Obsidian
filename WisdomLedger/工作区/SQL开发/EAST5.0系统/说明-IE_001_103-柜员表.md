# 说明-IE_001_103-柜员表

## 业务目标

- 从一表通柜员表（T_1_7）和机构信息表（T_1_1）映射生成 EAST5.0 柜员表（IE_001_103）。
- 报送截至采集日有效的柜员数据，以及上一采集日至采集日期间失效的终态数据。
- 包含实体柜员和虚拟柜员，通过"柜员类型"字段区分。

## 目标对象与数据粒度

- 目标对象：IE_001_103 柜员表
- 数据粒度：每个柜员一条记录（柜员号 + 内部机构号 + 采集日期）

## 使用的数据表及其角色

| 数据表 | 系统 | 角色 |
| --- | --- | --- |
| T_1_7 | 一表通系统 | 柜员明细主源 |
| T_1_1 | 一表通系统 | 机构信息补充表（金融许可证号、银行机构名称） |
| IE_001_103 | EAST5.0系统 | 目标输出表 |

## 主连接键与时间条件

- 柜员主源连接键：T_1_7.A070001（机构ID）LEFT JOIN T_1_1.A010001（机构ID）
- 时间条件：T_1_7.A070011（采集日期）= p_data_date 格式化后的 DATE
- 过滤条件：T_1_7.A070008（柜员状态）= '01'（在岗）OR T_1_7.A070009（失效日期）>= 当月月初

## 关键过滤条件

- 柜员状态 = '01'（在岗）：保留所有在岗柜员
- 失效日期 >= 当月月初：保留当月内失效的柜员（终态数据）
- 机构信息表去重：使用 NOT EXISTS 派生表取最新机构记录

## 码值转换说明

- 是否实体柜员 (SFSTGY)：`1 -> '是'`，`0 -> '否'`
- 柜员权限级别 (GYQXJB)：`01 -> '高柜'`，`02 -> '低柜'`，`00-*** -> '其他-***'`（前缀替换）
- 柜员状态 (GYZT)：`01 -> '在岗'`，`02 -> '离岗'`，`00-*** -> '其他-***'`（前缀替换）

## 无法确认的假设和风险

1. 一表通柜员表 T_1_7 的机构ID字段（A070001）是否所有记录均符合"第12位开始截取生成内部机构号"的编码规范，需与业务确认。
2. 柜员状态码值 01=在岗、02=离岗，'00-***'→'其他-***'，是否还有其他枚举值待补。
3. 柜员权限级别码值 01=高柜、02=低柜，'00-***'→'其他-***'，是否还有其他枚举值待补。
4. 上岗日期为 NULL 的虚拟柜员，当前未实现默认日期兜底规则。
5. "柜员状态转为注销、无效等时，可在报送该条数据最终状态的次月不再报送"规则，当前 SQL 中注销/无效是否对应柜员状态=02（离岗）仍待确认。
