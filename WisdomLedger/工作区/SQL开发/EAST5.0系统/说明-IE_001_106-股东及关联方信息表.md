## 实现口径

### 数据粒度

一条记录 = 一个股东或关联方ID（A060001）+ 机构ID（A060002）在某一采集日期（A060024）的报送记录。

### 报送范围

- **上市银行**（上市标识 A060029='1'）：持股比例 ≥ 1% 或 是否驻派董监事（A060022）='1'
- **未上市银行**（上市标识 A060029='0'）：全量报送
- **上市标识为空或其他值**：保守保留（待与业务确认）
- **终态排除**：当月状态=无效(00) 且 上月状态也为无效(00) → 不报送

### 关键关联

| 关联 | 源表 | 目标表 | 关联条件 | 用途 |
|------|------|------|----------|------|
| 机构信息 | T_1_6 | T_1_1 | A060002 = A010001 + 采集日期 | 取金融许可证号、银行机构名称 |
| 上月状态 | T_1_6 (cur) | T_1_6 (prev) | A060001 = A060001 + 上月最后一天 | 判断终态（无效延续） |
| 代码映射 | T_1_6 | dim_code_mapping | A060005 = src_code, rule=YBT-EAST-GDHGLFZJLB | 证件类别码值转换 |
| 对公客户映射 | T_1_6 | v_ybt_to_east_corp_cust | A060001 = src_cust_id | 获取客户统一编号 |
| 个人客户映射 | T_1_6 | v_ybt_to_east_person_cust | A060001 = src_cust_id | 获取客户统一编号 |

### 字段映射覆盖

27 个一表通映射字段 + 2 个 DDL 额外字段（GSFZJG、SENSITIVEFLAG），共 29 字段，全部覆盖。

## 待确认问题

| 序号 | 问题 | 影响 | 建议 |
|------|------|------|------|
| 1 | `dim_code_mapping` 代码映射表的物理表名、库归属和字段名需确认 | 证件类别转换逻辑当前依赖此表 | 确认后替换 SQL 中的表名和字段名 |
| 2 | `v_ybt_to_east_corp_cust` / `v_ybt_to_east_person_cust` 的物理表名/视图名和字段名需确认 | 客户统一编号映射当前依赖此表 | 确认后替换，或改为直接关联 T_2_1/T_2_5 |
| 3 | 一表通 T_1_6 的采集日期字段 A060024 是 date 类型，与 p_data_date(VARCHAR(8)) 的比较方式需确认 | 当前使用 `A060024 = v_data_date` | 若 T_1_6 实际存储格式不同需调整 |
| 4 | 内部机构号截取规则"第12位开始截取至最后一位"是否对所有机构ID恒成立 | 截取异常会导致空值 | 建议抽样验证机构ID长度和格式 |
| 5 | 归属分支机构（GSFZJG）的来源 | 当前按 NBJGH 兜底，可能不准确 | 与业务确认是否有独立来源字段 |
| 6 | 涉密标志（SENSITIVEFLAG）的来源 | 当前置空 | 与业务确认报送规则 |
| 7 | 股东或关联方状态(A060017)除 01/00 外是否存在其他枚举值 | 影响终态判断和状态码值转换 | 确认完整枚举 |
| 8 | 持股比例(A060019)是小数（如 0.05=5%）还是百分制（如 5=5%） | 影响上市分支 1% 阈值比较 | 当前假定为小数 |
| 9 | 上市标识(A060029)是否只有 '0'/'1' 两个值 | 影响分支逻辑 | 当前对非 0/1 值保守保留 |

## 运行建议

1. 先执行校验 SQL 第 1 节（行数对照），确认目标表行数与源表 eligible 行数一致。
2. 执行校验 SQL 第 2 节（主键重复检查），确认无重复。
3. 执行校验 SQL 第 4 节（码值越界检查），确认码值转换正确。
4. 执行校验 SQL 第 8 节（分支逻辑校验），确认上市/未上市过滤正确。
