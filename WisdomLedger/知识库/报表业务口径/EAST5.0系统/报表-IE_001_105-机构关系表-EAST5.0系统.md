# 报表-IE_001_105-机构关系表-EAST5.0系统

## 系统归属

- 所属系统：EAST5.0系统
- 数据表编号：105
- 数据表代码：JGGXB
- 主题：公共信息

## 报送概述

- 报送模式：全量表，报送截至采集日有效的数据，以及上一采集日至采集日期间结清、失效、终结等所有视为终态的数据。
- 报送范围：应包括所有进行账务处理的机构，至少应包括总行、分行、支行等实体机构和省级汇总、地市级汇总、法人汇总、分行汇总。会计全科目表中的机构在机构关系表中都需要报送。
- 每次全量报送。

## 整表口径

- 统计对象：机构与其上级管理机构的管理关系。
- 统计粒度：每个内部机构号 + 采集日期一条记录。
- 纳入规则：
  - 取自一表通机构关系表（T_1_2）当日采集日期（A020003）的所有记录。
  - 机构ID与机构信息表（T_1_1）通过机构ID关联，获取机构名称、支付行号、许可证号等属性。
- 排除规则：当前未定义显式排除规则。
- 时间口径：采集日期（A020003），格式 YYYYMMDD。

## 纳入排除规则

### 整表纳入

- 一表通机构关系表（T_1_2）当日采集日期（A020003 = p_data_date）的全部记录。

### 无显式排除

- 当前未定义特殊的机构排除规则。如需排除特定机构类型或状态，应在 SQL 的 WHERE 子句中补充。

## 关键业务规则

- 内部机构号：由机构ID从第12位截取，机构ID格式为"前11位+内部机构号"。
- 上级管理机构：当上级管理机构ID='0'时，表示该机构为顶层机构，上级管理机构代码和上级管理内部机构号均填'0'，上级管理机构名称填本机构名称。
- 采集日期：从一表通的 date 类型字段格式化为 YYYYMMDD 字符串。
- 去重：T_1_1 按（A010001, A010020）去重，每机构每天只保留一条，避免关联一对多产生重复。
