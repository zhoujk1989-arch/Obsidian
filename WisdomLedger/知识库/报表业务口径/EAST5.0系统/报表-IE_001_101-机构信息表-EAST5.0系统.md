## 业务定义

- 这张报表回答的业务问题：EAST5.0 需要报送哪些银行机构，以及机构代码、名称、类别、状态、地址、负责人、联系电话等信息是什么。
- 统计对象：一表通 `1.1 机构信息` 中满足 EAST 映射规则的机构记录。
- 统计时点：过程入参 `I_DATE`，写入 EAST 字段 `CJRQ`，格式为 `YYYYMMDD`。

## 整表报送范围

- 主来源为一表通 `T_1_1` 当日机构信息快照。
- 以 `T_1_1.A010002` 内部机构号去重，每个内部机构号保留一条记录。
- 左关联上月末 `T_1_1`，识别上月已停业且当月仍停业的机构。
- 左关联一表通员工表 `T_1_3`，用负责人内部工号取负责人职务。
- 用一表通总账会计全科目 `T_4_1` 和分户账信息 `T_4_3` 识别仍有余额或仍存在分户账的机构，作为停业机构保留的回补依据。

## 纳入与排除规则

- 纳入：
  - 当日 `T_1_1.A010020 = I_DATE` 的机构快照。
  - 上月与当月均停业，但在总账会计全科目余额不为 0 或内部分户账中仍出现的机构。
- 排除：
  - 上月末为停业、当月仍为停业，且无总账余额或分户账回补证据的机构。
- 当前假设：
  - 图片中的“停业和被合并”在当前一表通字段中按 `A010014 in ('00','02','03')` 识别；若后续补到合并状态专门码值，应同步调整 SQL 和本页。

## 字段或指标特殊规则

- `JGLB` 是 EAST 中文枚举，不能直接沿用一表通数字代码。
- `YYZT` 是 EAST 中文枚举，当前将一表通 `01` 识别为营业，其余停业类代码识别为停业。
- `CLRQ`、`CJRQ` 均输出为 `YYYYMMDD`。
- `FZRZW` 不是机构表直接字段，需要通过负责人工号关联员工表。
- `GSFZJG`、`SENSITIVEFLAG` 在用户映射图中未给出来源，当前脚本置空并保留缺口。
- 原始 SQL 与工作区草案仅调整 SQL 组织方式，不改变当前业务筛选、字段映射和缺口字段处理。
