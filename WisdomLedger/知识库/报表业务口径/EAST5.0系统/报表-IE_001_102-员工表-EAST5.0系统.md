## 业务定义

- 这张报表回答的业务问题：EAST5.0 需要报送哪些员工，以及员工所属机构、身份、证件、岗位、高管标识、员工类型和状态是什么。
- 统计对象：一表通 `T_1_3` 员工表中截至采集日在岗、待岗，或当月离职的员工。
- 统计时点：过程入参 `p_data_date`，写入 EAST 字段 `CJRQ`，格式为 `YYYYMMDD`。

## 整表报送范围

- 主来源为一表通 `T_1_3` 员工表。
- 关联一表通 `T_1_1` 机构信息表，补充金融许可证号和银行机构名称。
- 关联一表通 `T_10_1` 公共代码表，按 `通用/国家地区` 和 `通用/证件类型` 转换员工国籍和证件类别。
- 按 `员工ID + 机构ID` 去重，避免同一员工同机构插入多条。

## 纳入与排除规则

- 纳入：
  - `T_1_3.A030028 = p_data_date` 的员工快照。
  - 员工状态为 `01` 在岗或 `03` 待岗。
  - 或离职日期 `A030029` 大于等于当月月初。
- 排除：
  - 不满足上述状态或离职日期条件的员工。
  - 子公司员工不用报送；当前 SQL 尚无子公司标识来源，暂未实现该排除。

## 字段或指标特殊规则

- 员工状态过滤：只报在岗、待岗，以及本月离职人员最后状态。
- 在职类员工需每月报送；离职员工在报送最后状态的次月可不再报送。
- 子公司员工不用报送，理财子公司应报送；当前没有稳定字段实现该过滤，保留为缺口。
- `NBJGH` 不直接取机构信息表内部机构号，而是按映射清单从员工 `机构ID` 截取。
- 原始 SQL 与工作区草案仅调整 SQL 组织方式，不改变当前业务筛选、字段映射和缺口字段处理；员工主源按实际使用字段 `distinct` 消除完全重复行。
