# 报表-1.1-机构信息-一表通系统

## 系统归属

- 所属系统：一表通系统
- 是否跨系统汇总：是
- 如果跨系统，分别涉及哪些系统：一表通系统、监管集市系统
- 文件路径是否位于正确系统目录：是

## 报表类型

- 报表分类：明细类
- 报送形态：按 `采集日期 + 机构号` 输出机构明细记录
- 本页写法：按整表报送范围和字段级口径展开，不以汇总指标为主

## 业务定义

- 这张报表回答的业务问题是什么：截至采集日，一表通需要报送哪些机构，以及这些机构的身份、层级、状态、区划和负责人信息分别是什么。
- 统计对象是谁：纳入口径内的银行机构、管理机构、营业机构、虚拟机构和内设机构。
- 统计时点/周期是什么：按 `I_DATE` 生成日报快照，结果落入 `T_1_1`。

## 统计口径

- 纳入口径：
  - 以 `SMTMODS.L_PUBL_ORG_BRA` 当日快照为基础。
  - 允许保留当前失效、但上年末或本年内曾有效的机构。
- 排除口径：
  - `ORG_NUM = '999999'`
  - `ORG_NUM` 以 `5`、`6`、`7` 开头
  - 一批指定机构号和瑞祥下属村镇机构号
- 统计粒度：`采集日期 + 机构号`
- 输出维度：
  - 机构身份与名称：`A010001~A010008`
  - 县域/科技/区划/状态：`A010009~A010014`
  - 成立日期、地址、负责人：`A010015~A010019`
  - 采集日期与新增字段：`A010020~A010023`

## 整表报送范围

- 报送对象：
  - 监管集市机构表 `L_PUBL_ORG_BRA` 中满足纳入口径的机构快照记录
  - 包括总行、分行管理机构、营业机构、虚拟机构和内设机构
- 报送目的：
  - 向一表通提供机构基础信息主档，供监管机构识别机构身份、区划、状态和负责人
- 明确不报对象：
  - `ORG_NUM = '999999'`
  - `ORG_NUM` 以 `5`、`6`、`7` 开头的机构
  - 指定剔除机构号清单中的机构
- 报送模式：
  - 日截面全量报送
  - 当天先删后插，以 `A010020=采集日期` 为重算边界

## 字段或指标特殊规则

- 名称修饰规则：
  - `ORG_NUM='990000'` 不改名
  - `ORG_NUM='000000'` 固定显示 `吉林银行股份有限公司(总行)`
  - `ORG_NUM` 末四位 `0000` 补 `(分行管理机构)`
  - 部分末两位 `00` 且满足条件的机构补 `(管理机构)`
- 同名机构规则：
  - 相同 `computed_name` 下，按成立日期最早的一条追加 `(待撤销)`
- 失效保留规则：
  - 当前失效但上年末有效，保留
  - 当前失效但本年内曾有效，保留
- 排除规则：
  - 显式剔除瑞祥下属村镇机构和其他指定机构号
- 版本差异：
  - 外部数据字典已有 `A010024~A010026`，当前过程未覆盖
