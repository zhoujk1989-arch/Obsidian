# 概念-系统-EAST5.0系统

## 系统定位

- EAST5.0系统是监管数据采集系统，本仓库当前把 `IE_...` 表作为独立系统对象维护。
- EAST5.0 表与一表通 `T_...` 表默认互相独立；跨系统映射通过血缘页、报表业务口径页和来源页表达，不共享路径。
- 原始 DDL 存放在 `原始材料/表结构/EAST5.0系统/`，SQL 草案存放在 `工作区/SQL开发/EAST5.0系统/`，正式本地 SQL 存放在 `原始材料/SQL/EAST5.0系统/`。

## 维护规则

- 字段含义、主键、隐私变形、默认值和码值说明写入 `知识库/数据表/EAST5.0系统/`。
- 报表是否报送、报送范围、纳入排除规则写入 `知识库/报表业务口径/EAST5.0系统/`。
- SQL 草案未运行验证前，报表页、数据表页、来源页和血缘页通常保持 `draft`。
- 只有需要字段来源、SQL 赋值、过滤条件或跨系统链路时，才读取 `知识库/血缘/EAST5.0系统/`。
