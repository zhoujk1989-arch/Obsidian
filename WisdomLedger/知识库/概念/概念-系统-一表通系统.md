# 概念-系统-一表通系统

## 定义

- 一表通系统是 WisdomLedger 中独立维护的监管报送系统边界。
- 该系统下的 SQL、数据表、来源页、报表页和血缘页应分别放在 `一表通系统` 对应目录下维护。
- 一表通结果表通常以 `T_<报表编号>` 形式命名，例如 `T_1_1`、`T_1_2`。

## 当前状态

- 已建立 `1.1 机构信息` 的来源页、报表页、数据表页和血缘页。
- 已初始化 `1.2 机构关系` 的来源页和数据表页；当前仅有 DDL 证据，仍缺外部监管来源和加工 SQL。
- 已初始化 `1.3 员工` 的来源页和数据表页；当前仅有 DDL 证据，仍缺外部监管来源、加工 SQL 和代码类字段码值说明。
- 一表通接口表或结果表中的代码类字段，应直接写入对应数据表页字段口径，不再拆成独立码值页。

## 维护规则

- 新增一表通表结构时，先按系统目录创建或更新来源页与数据表页。
- 只有在存在报表 SQL、存储过程、视图定义或明确上下游证据时，才创建或扩展血缘页。
- 报表字段是否为监管要求字段，应优先依据 `my-dev-brain` 中的填报说明、实体页或原文页；不能仅凭 DDL 判断。
- 字段是否已经加工落地，应同时看目标表结构和 SQL 赋值证据；不能仅凭外部说明或 DDL 判断。
