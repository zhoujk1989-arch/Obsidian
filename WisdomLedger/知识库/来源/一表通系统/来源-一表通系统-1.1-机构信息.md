## Key Findings

- 该过程按 `I_DATE` 先删除 `T_1_1` 当日快照，再重新插入机构信息结果。
- 统计对象来自 `SMTMODS.L_PUBL_ORG_BRA` 当日机构快照，并通过多级 `UP_ORG_NUM` 回溯补全上级机构证照和编码。
- 明确排除了 `999999`、以 `5/6/7` 开头的机构号，以及一批指定机构号。
- 失效机构并非全部剔除，若该机构在上年末或当年内曾存在有效状态，仍会保留。
- `my-dev-brain` 2026-04-27 填报说明和实体页显示 `1.1机构信息` 当前为 26 个字段，2.1 版新增 `A010024` 机构联系电话、`A010025` 本年累计科技投入金额、`A010026` 备注。
- 2026-04-28 本地 DDL 已包含 `A010001~A010026`，主键为 `A010001 + A010002 + A010020`。
- 当前本地存储过程 `PROC_BSP_T_1_1_JGXX` 的 `INSERT INTO T_1_1` 仍只映射 `A010001~A010023`，未给 `A010024~A010026` 赋值；因此这三个字段当前只能确认“监管要求存在、表结构存在”，字段级血缘和取数规则仍待补。
- 后续 agent 判断“一表通需要哪些字段”时，应先以外部实体页和填报说明确定监管要求字段，再用本地 DDL 判断目标表是否已具备字段，最后用 SQL 血缘判断字段是否已经实际加工落地；不能仅凭 DDL 或 SQL 单独下结论。

## Evidence

```sql
DELETE FROM T_1_1 WHERE A010020 = to_char(P_DATE,'yyyy-mm-dd');

INSERT INTO T_1_1 (...)
SELECT ...
FROM SMTMODS.L_PUBL_ORG_BRA T1
LEFT JOIN SMTMODS.L_PUBL_ORG_BRA T2 ON T1.UP_ORG_NUM = T2.ORG_NUM
...
WHERE T1.DATA_DATE = I_DATE
  AND T1.ORG_NUM <> '999999'
  AND T1.ORG_NUM NOT LIKE '5%'
  AND T1.ORG_NUM NOT LIKE '6%'
  AND T1.ORG_NUM NOT LIKE '7%';
```

```sql

  `A010026` varchar(600) DEFAULT NULL COMMENT '备注；原始数据格式：anc..600',
  PRIMARY KEY (`A010001`, `A010002`, `A010020`)
);
```
